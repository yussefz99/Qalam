<objective>
Fix the two device bugs root-caused this session: (1) **thaa is always wrong** — the
device reads letters Firestore-first, and neither the prod Firestore `thaa` doc nor the
bundle carries per-form `contextualForms`, so the scorer has no per-form reference to
score thaa against; (2) **silent fail** — every thaa fail resolves to no authored line
and renders/speaks an empty string.

Four moves, in order, on the MAIN working tree:
1. Surgically merge ONLY `contextualForms` for alif/baa/taa/thaa from the owner's
   untracked `letters (2).json` into `assets/curriculum/letters.json` (keep the app's
   current `signedOff` values; do NOT take alif `commonMistakes`; do NOT touch the other
   24 letters; do NOT commit the owner source file).
2. Add a warm generic tutor-voice floor line so a scorer FAIL that resolves no authored
   per-mistake line is never silent — in BOTH `_floorLineFor` (verdict/voice side) and
   `AuthoredFallbackBrain._resolveLine` (offline brain side).
3. Author per-mistake fail lines for the 19 thaa exercises, keyed to mistakeIds the
   scorer can actually emit for each exercise type (derived from the validator, not
   invented). All stay `signedOff: false` for the mother's review packet.
4. As a LATE, single, owner-authorized PROD write: re-seed prod Firestore `letters` from
   the corrected bundle, then verify `letters/thaa` carries `contextualForms` via REST.

Purpose: thaa scores against real per-form references (no longer always-wrong) and every
fail speaks a grounded, warm line — on-device AND offline.

Output: merged bundle, authored thaa fail lines, non-empty feedback floor, a re-seeded
prod Firestore, and a mother's-review list of every authored line in the SUMMARY.
</objective>

<verified_findings>
Confirmed against the code + live prod Firestore this planning session:

- **Firestore-first read** (`curriculum_repository.dart:105` runs the load-time guard over
  whichever source won). Prod `letters/thaa` (project `qalam-app-bd7d0`) has NO
  `contextualForms` field — verified via REST readback (fields present: audio, char,
  cleanRepsToAdvance, commonMistakes, forms, id, introOrder, mistakesStatus, name,
  referenceStrokes, signedOff, tolerances). So the device scores thaa with no per-form
  reference → always wrong.
- **Bundle diff** (bundle vs `letters (2).json`, both keyed by id):
  - `thaa`: bundle `contextualForms: null`; owner has all four forms. **Take owner
    contextualForms.** (bundle signedOff=false, owner signedOff=false — no signedOff conflict)
  - `baa`: both have four forms; **owner signedOff=false, bundle signedOff=true.** Take
    owner contextualForms ONLY; KEEP bundle signedOff=true (the mother signed baa).
  - `taa`: same as baa — owner signedOff=false, bundle signedOff=true. Take owner
    contextualForms ONLY; KEEP bundle signedOff=true.
  - `alif`: both signedOff=false. Take owner contextualForms. Owner also changes
    `commonMistakes` — **do NOT take that** (flag for the mother).
- **Silent-fail mechanism** (double-confirmed): `exercise_validator._genericMiss` returns
  the sentinel `'tryAgain'` when the exercise authored no non-`pass` key. `'tryAgain'` is
  NOT a feedback key, so `_floorLineFor` / `_resolveLine` look it up (null), then loop
  over non-`pass` entries (none), then return `''`. Empty line → silent + silent voice.
- **Scorer→feedback-key mapping** lives in `exercise_validator._mapMistake`
  (`candidatesByMistake`): the scorer's `MistakeId` → an ordered candidate list; the FIRST
  key the exercise authored wins, else `_genericMiss`. So authoring a candidate key makes
  it fire. The real per-type candidate sets:
  - glyph (traceLetter isolated/initial/medial): geometry ids →
    `shallowBowl` (tooShort/tooCurved/dotMisplaced), `noDot`/`missingDot`/`dotMisplaced`
    (wrongStrokeCount/wrongStrokeOrder/dotMisplaced), `wrongStrokeCount`, `wrongStrokeOrder`.
    thaa is THREE dots above → dot lines are the highest-value.
  - glyph+positionalForm (writeLetter): `wrongLetter`, `wrongForm`, plus the geometry keys.
  - sequence (writeWord / completeWord / fillBlank): `missingDot`, `incomplete`, `wrongWord`.
  - sequence+joinContinuity (connectWord): `lifted` (authored first), then `incomplete`.
  - order+sequence (buildSentence): `wrongOrder`, then `incomplete`.
  - sequence+transformRule (transformWord): `missingEnding`, `wrongForm`, `wrongAntonym`.
- **The `contextualForms` pipeline already works** end to end: `point_codec.encode_letter`
  encodes nested contextualForms strokes to `{x,y}` maps; the Dart codec/`Letter.fromJson`
  parse them. Only the DATA is stale/missing.
- **Seed auth**: `seed_firestore.py` needs `firebase_admin` (NOT importable here) +
  `GOOGLE_APPLICATION_CREDENTIALS` (unset; no service-account key on disk). `gcloud auth
  print-access-token` succeeds; gcloud project = `qalam-app-bd7d0`. Preferred path: pip
  install `tools/firebase/requirements.txt` into a venv and run the real script (it reuses
  `encode_letter`). REST PATCH is the documented fallback if install fails.
- Branch is `main`, tree clean apart from the untracked `letters (2).json` (per memory
  "phase-work-needs-own-branch" — this is an owner-scoped quick fix on the main working
  tree per locked scope; NOT phase/execution work, so no phase branch).
- **KNOWN pre-existing failures to IGNORE** (never fix / never re-bake): alif_reference
  cluster, all_letters_validation signedOff, reference_overlay golden, meet_section
  img.door, mastery_celebration golden, glyph_audit golden. The merged alif contextualForms
  MAY change alif_reference results — if those tests change status, REPORT it, do not chase.
- The load-time repo guard + all_letters_validation validate only TOP-LEVEL
  `referenceStrokes`, NOT nested contextualForms strokes. Task 1's test therefore validates
  the merged contextualForms strokes explicitly (closed-loop / range guard) as a safety net.
