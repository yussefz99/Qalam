## What was built

### Task 1 — Surgical contextualForms merge tool (code complete, NOT applied)

`tools/curriculum/merge_contextual_forms.py` — stdlib-only (json/pathlib/argparse),
repo-root via `Path(__file__).resolve().parents[2]`, `--check` self-verify affordance,
mirroring `tools/firebase/point_codec.py` conventions. It copies **ONLY** the
`contextualForms` field for exactly `alif, baa, taa, thaa` from the owner source into the
bundle — never `signedOff`, never `commonMistakes`, never `referenceStrokes`, never
`tolerances`. It errors naming any missing source id and skips+warns on a null source
`contextualForms`. Serialization is `json.dumps(indent=2, ensure_ascii=False)`, which I
VERIFIED round-trips the current bundle **byte-for-byte** (no trailing newline) via a
successful Bash probe earlier in the session — so the 24 untouched letters stay identical
and a re-run is a no-op (idempotent). `--check` exits 0 when the bundle already equals the
merged result, non-zero when the merge would still change it.

`test/curriculum/thaa_contextual_forms_merge_test.dart` (new) asserts, over the loaded
bundle: thaa carries all four `contextualForms` forms each with non-empty strokes; every
merged `contextualForms` stroke of alif/baa/taa/thaa passes `validateReferenceStrokes`
(the nested safety net — the load-time guard only covers top-level strokes); signedOff is
baa=true/taa=true/alif=false/thaa=false; alif.commonMistakes is the **bundle** value (3
mistakes: strokeLengthBelowThreshold / strokeDirectionInverted / strokeCurvatureExceedsThreshold),
NOT the owner file's.

**Verified by reading both files (Read tool):** owner `thaa` has all four
`contextualForms` forms (isolated/initial/medial/final) with valid strokes; owner and
bundle thaa `referenceStrokes` + `commonMistakes` are identical (so only `contextualForms`
needs to cross). Bundle thaa has NO `contextualForms` key at all (goes referenceStrokes →
cleanRepsToAdvance directly), so the merge APPENDS it as the last field, matching where
alif's `contextualForms` sits.
