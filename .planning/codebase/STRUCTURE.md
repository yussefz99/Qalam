## Directory Purposes

**`lib/`:**
- Purpose: All Flutter/Dart source code for the app
- Contains: Widgets, screens, services, models (not yet organized by feature)
- Currently: Only `main.dart` (boilerplate)
- Planned subdirs (not yet created):
  - `screens/` — Full-page widgets (practice lesson, progress dashboard)
  - `widgets/` — Reusable UI components (stroke canvas, feedback card, tutor response box)
  - `models/` — Data classes (Letter, Session, Stroke, Child, Lesson)
  - `services/` — Business logic (HandwritingService, TutorService, SessionRepository)
  - `providers/` — Riverpod state providers
  - `theme/` — App theme, colors, text styles
  - `utils/` — Helpers, constants, extensions

**`test/`:**
- Purpose: Dart/Flutter tests for the app
- Currently: Boilerplate widget test for counter demo
- Pattern: Co-located by feature (not yet established)
- Will grow to: Unit tests for models/services, widget tests for screens, integration tests for Firebase flows

**`android/`:**
- Purpose: Android native build and runtime integration
- `android/app/src/main/kotlin/` — Kotlin source (MainActivity stub)
- `android/app/src/main/java/` — Java source (currently empty; reserved for future native code)
- `android/app/src/main/res/` — Resources (app icon, colors, strings, dimensions)
- `android/app/build.gradle.kts` — App build configuration (SDK versions, dependencies, signing)
- `android/gradle/wrapper/` — Gradle wrapper (version pinned, not to be edited manually)
- Build output: `android/app/debug/`, `android/app/release/`, `android/app/profile/` (in `.gitignore`)

**`docs/`:**
- Purpose: Project documentation and research findings
- `docs/RESEARCH_BRIEF.md` — Open research questions that gate implementation (R1–R5)
- Planned subdirs (from CLAUDE.md):
  - `docs/research/raw/` — Raw findings per research question (one file per question)
  - `docs/architecture/` — Compiled decisions & ADRs (architecture decision records)

**`.planning/codebase/`:**
- Purpose: Automated codebase analysis documents (ARCHITECTURE.md, STRUCTURE.md, etc.)
- Generated by: `/gsd-map-codebase` command
- Consumed by: `/gsd-plan-phase` and `/gsd-execute-phase` commands

**`.claude/`:**
- Purpose: Claude AI tooling, agent definitions, GSD workflows
- `agents/` — Subagent plugin configurations (Flutter architect, state management, Firebase, UI, testing)
- `commands/gsd/` — GSD command implementations
- `get-shit-done/` — Workflow templates, contexts, references for GSD phases

## Key File Locations

**Entry Points:**
- `lib/main.dart`: Flutter app entry; `void main()` and root widget `MyApp`

- `pubspec.yaml`: Dart/Flutter dependency manifest and asset configuration
- `pubspec.lock`: Locked dependency versions (auto-generated, committed to git)

- `android/app/build.gradle.kts`: Android app build (SDK, signing, plugin configuration)
- `android/app/src/main/AndroidManifest.xml`: Android app permissions and activity declaration
- `CLAUDE.md`: Project context, constraints, and architectural decisions

**Documentation:**
- `docs/RESEARCH_BRIEF.md`: Open research questions (mandatory before dependent code)
- `README.md`: Basic project overview (placeholder)

**Project Metadata:**
- `.metadata`: Flutter metadata (SDK version, platform info)
- `.gitignore`: Files excluded from git
- `qalam.iml`: IntelliJ project file (IDE config, auto-generated)

## Naming Conventions

**Files:**
- **Dart/Flutter:** `snake_case` (e.g., `handwriting_service.dart`, `stroke_model.dart`)
- **Kotlin/Java:** `PascalCase` for classes (e.g., `MainActivity.kt`)
- **Resources (Android):** `snake_case` (e.g., `ic_launcher.png`, `strings.xml`)

**Directories:**
- **Dart packages/features:** `snake_case` (e.g., `lib/services/`, `lib/screens/`)
- **Android package structure:** Reverse-domain (e.g., `com.example.qalam`)

**Classes:**
- **Dart:** `PascalCase` for all classes (e.g., `Letter`, `Session`, `TutorService`)
- **Kotlin:** `PascalCase` for classes (e.g., `MainActivity`)

**Functions/Methods:**
- **Dart:** `camelCase` for functions and methods (e.g., `captureStroke()`, `validateLetter()`)

**Constants:**
- **Dart:** `camelCase` or UPPER_SNAKE_CASE for top-level (convention varies; typically `camelCase` for final variables)

## Where to Add New Code

**New Feature (e.g., "Practice Lesson Screen"):**
- Primary code: `lib/screens/practice_screen.dart`
- State management: `lib/providers/practice_provider.dart` (Riverpod)
- Models: `lib/models/lesson.dart`, `lib/models/stroke.dart`
- Tests: `test/screens/practice_screen_test.dart`, `test/providers/practice_provider_test.dart`

**New Component/Widget:**
- Reusable UI: `lib/widgets/stroke_canvas.dart`, `lib/widgets/feedback_card.dart`
- Screen-specific: Keep within `lib/screens/` folder or as a module (e.g., `lib/screens/practice/widgets/`)
- Tests: `test/widgets/stroke_canvas_test.dart`

**Service/Business Logic:**
- Handwriting recognition wrapper: `lib/services/handwriting_service.dart`
- Tutor interaction: `lib/services/tutor_service.dart`
- Firebase/data access: `lib/services/firebase_service.dart` or feature-specific `lib/repositories/session_repository.dart`

**Utilities:**
- Shared helpers: `lib/utils/constants.dart`, `lib/utils/extensions.dart`
- Validators: `lib/utils/validators.dart`
- Formatters: `lib/utils/formatters.dart`

**Models/Domain:**
- Core data classes: `lib/models/letter.dart`, `lib/models/session.dart`, `lib/models/stroke.dart`
- Enums: `lib/models/lesson_status.dart`

**Theme/Styling:**
- App theme: `lib/theme/app_theme.dart`
- Colors: `lib/theme/colors.dart`
- Text styles: `lib/theme/text_styles.dart`

**Android/Native Integration:**
- Native features: `android/app/src/main/kotlin/com/example/qalam/` (subdirs by feature, e.g., `stylus/`, `mlkit/`)
- Resource overrides: `android/app/src/main/res/` (colors, strings, dimensions specific to Qalam)
