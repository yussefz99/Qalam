# Phase 02.1.1 Plan 04: Trace Screen + DottedGuidePainter Summary

**The half-traced alif: a reusable DottedGuidePainter renders the dotted reference guide plus a smoothed deep-ink overlay over the first 50% of the stroke (both from the single DemoAlif source), and DemoTraceScreen frames it with a neutral "Stroke 1 of 1" and a demoTraceSubmit CTA that leads to the HERO Feedback miss screen — with no recognition/capture engine.**

## Performance

- **Duration:** ~22 min
- **Started:** 2026-06-01
- **Tasks:** 2/2 (both TDD: RED → GREEN)

## What shipped

### Task 1 — DottedGuidePainter (reusable, Pitfall 5)
- `lib/demo/widgets/dotted_guide_painter.dart`: a `CustomPainter` taking normalized `referencePoints` + an `inkProgress` (0..1). Layer 1 is the full dotted guide (dashed via `PathMetric.extractPath`, muted `QalamColors.fgMuted` — never gold/coral). Layer 2 is a quadratic-smoothed ink overlay (`QalamColors.inkStroke`, `QalamInk.strokeWidth`, round caps/joins — the Practice treatment) over the first `inkProgress` fraction by arc length.
- Pure geometry helpers `scaledGuidePoints(size)` and `inkPath(size)` make the half-traced state unit-testable: 4/4 tests assert no-throw paint, ink-length scaling with progress (0 → none, 0.5 → ~half, 1.0 → full), normalized→pixel mapping, and `shouldRepaint` value-equality.

### Task 2 — DemoTraceScreen
- `lib/demo/screens/demo_trace_screen.dart`: parchment Scaffold with the "YOUR TURN · TRACE" eyebrow, "Now You Trace Alif." heading, a quiet "Stroke 1 of 1" label (information, not a score — DP-06), the soft-aqua/parchment canvas card painting `DottedGuidePainter(inkProgress: 0.5)`, and a keyed sticker-shadow submit CTA labeled by `demoTraceSubmit` → `/demo/feedback`.
- Wired `/demo/trace` → `DemoTraceScreen` in the `_screenFor` dispatch.
