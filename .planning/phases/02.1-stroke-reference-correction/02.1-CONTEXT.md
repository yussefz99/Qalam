# Phase 02.1: Stroke Reference Correction — Context

**Gathered:** 2026-06-01
**Status:** Ready for planning
**Source:** Decisions captured in the planning session that surfaced the outline-vs-centerline defect (research: `.planning/research/STROKE-REFERENCE.md`).

<domain>
## Phase Boundary

A focused **corrective amendment** between Phase 2 and Phase 3. Phase 2 shipped
`referenceStrokes` as glyph **outline contours** (closed silhouette loops) instead of
the **open centerline** a pen tip travels — breaking both the Phase 3 geometric scorer
and the "watch me write" animation. This phase fixes the representation, the tooling
that produced it, and the one signed-off letter (alif), and installs a guard so the
bug cannot return.

**In scope:**
- Data-model change: add a `type` field to `StrokeSpec`; validate `direction` against point order.
- A load-time / test validator that rejects closed-loop / outline strokes.
- Repurpose the Python extraction tool so it can never again emit outlines as teaching strokes.
- An in-app **authoring trace screen** (owner-facing dev tool) to capture correct strokes.
- Re-author **alif** correctly as the proven exemplar, with owner re-sign-off.
- A per-letter visual-overlay golden as the sign-off + regression artifact.

**Out of scope (deferred):**
- Authoring the other 27 letters — they stay `signedOff: false` placeholders, authored at Phase 7's existing sign-off gate.
- The SVG-fallback ingester — build only if a looped letter later needs it.
- Any scorer / animation implementation — that is Phase 3.
</domain>

<decisions>
## Implementation Decisions

### Phasing
- **D-01** — The fix lives in **Phase 02.1** (a focused amendment), not folded into Phase 3 and not a full 28-letter re-author. The other 27 letters are deferred to Phase 7's sign-off gate. *(Owner decision.)*

### Authoring method
- **D-02** — Authoring is an **in-app trace screen**: the owner (with his mother) traces each stroke over a faint Noto Naskh glyph, in her prescribed order and direction; reuse the Phase-1 `Listener` + `CustomPainter` capture (`lib/screens/practice_screen.dart`). Works finger-only (owner has no stylus yet). SVG vector authoring is a documented fallback, **not built now**. *(Owner decision.)*

### Data model
- **D-03** — Add `type: "line" | "curve" | "dot"` to `StrokeSpec` with a safe default (`json['type'] as String? ?? 'line'`) so existing data still parses. Keep `points` as normalized 0..1 `List<List<double>>`. Keep `direction`, but treat the **points as the source of truth** and make `direction` a validated, human-readable hint. A dot is `type:"dot"`, `points:[[x,y]]` (one point), `direction:"tap"`. *(Owner-approved.)*
- **D-04** — A pure-Dart **validator** (run at load and in tests) asserts, per non-dot stroke: not a closed loop (`distance(first,last) > ε` AND total length not ~2–3× the bbox diagonal); `direction` agrees with `first→last` order; coords in [0,1]; `order` 1..N contiguous with dots after body strokes; `type:"dot"` ⇒ exactly one point. This single guard is what would have caught the original bug. *(Owner-approved.)*

### Tooling
- **D-05** — Repurpose `tools/extract_reference_paths.py`: it must **never** emit body outlines as `referenceStrokes`. Keep only its dot-centroid + glyph-bbox output as **authoring hints** (rename output to `tools/authoring_hints.json`) and add a loud deprecation header pointing to `.planning/research/STROKE-REFERENCE.md`. *(From research §7.1.)*

### Alif (the exemplar — touches signed-off curriculum)
- **D-06** — Re-author **alif** as an open top→bottom centerline (research §7.3 candidate: a near-vertical line `[[0.5,0.0] … [0.5,1.0]]`, `type:"line"`). ⚠️ alif is `signedOff: true` today with wrong data; correcting it **re-opens signed-off curriculum**, so the owner confirms the exact start-x / Naskh lean **with his mother** and re-sets `signedOff: true` only after the visual-overlay confirmation. Alif's three `commonMistakes` stay as-is (they become valid against the corrected centerline). *(Owner pedagogy — explicit re-sign-off required.)*

### Sign-off artifact
- **D-07** — A **visual-overlay golden** per authored letter (the pen path drawn over the faint glyph, animated in draw order) is the human sign-off artifact AND the CI regression guard. *(From research §7.5.)*

### Pedagogy boundary
- **D-08** — Per-letter pen motion for ambiguous letters (seen/sheen teeth as one stroke or several; taa_h/zhaa/kaaf uprights as separate strokes; multi-dot marks as one mark or several) is the **owner's mother's domain** and is NOT decided here — it only matters when those letters are authored (Phase 7). *(Pedagogy boundary, CLAUDE.md.)*

### Claude's Discretion
- Test file structure and paths; the exact validator epsilon and length thresholds; how the authoring screen serializes/export the JSON fragment; the dev authoring tool's widget layout (it is an internal tool, not child-facing brand UI). Keep Dart changes small and explainable (owner is new to Dart).
</decisions>
