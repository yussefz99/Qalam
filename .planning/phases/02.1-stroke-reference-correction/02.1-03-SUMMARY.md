# Phase 02.1 Plan 03: Stroke-Reference Tooling (Extractor Repurpose) Summary

**Repurposed `tools/extract_reference_paths.py` so it can never again emit a glyph body outline as a teaching stroke — it now emits only dot centroids + glyph bbox as authoring hints to `tools/authoring_hints.json`, pinned by a dependency-free Python check.**

## Performance

- **Duration:** ~8 min
- **Tasks:** 2 completed
- **Files modified:** 3 (1 modified, 2 created)

## Accomplishments

### Task 1 — Repurpose the extractor (D-05)
- Removed the entire body-outline polyline pipeline (`quadratic_bezier_polyline`, `normalize_contours`, the `contours_out` emission). The tool no longer has any code path that produces a body-outline polyline.
- Added `separate_dots_and_bbox()`: identifies the body as the largest-bounding-box contour, discards it, and emits each remaining contour as a normalized dot centroid (0..1 in the shared per-letter bbox), plus the glyph bbox.
- Repointed `OUTPUT_PATH` from `tools/candidate_paths.json` to `tools/authoring_hints.json`. The old body-outline output is now dead; the rename stops anyone pasting outlines into `letters.json` again.
- Prepended a loud DEPRECATION header (and rewrote the module docstring + `Usage`) stating outlines are NOT teaching strokes, that the body centerline is owner-authored in the in-app trace screen, and pointing to `.planning/research/STROKE-REFERENCE.md`.
- Kept the graceful `fonttools`-absent exit and the composite-glyph resolution logic.
- Commit: `ca67a24`

### Task 2 — Pin the invariant + ship the hints file
- Added `tools/test_authoring_hints.py` (run `python3 tools/test_authoring_hints.py`, exit 0 on pass). It is dependency-free: imports the extractor without running `main()` and exercises `separate_dots_and_bbox` on synthetic contours.
  - Asserts static guarantees: output renamed to `authoring_hints.json`, deprecation header present, `OUTPUT_PATH` no longer targets `candidate_paths.json`.
  - Asserts behavior: a body + two dot contours yields exactly two normalized centroids (body discarded), the result is not a polyline (≥4 [x,y] pairs), the bbox is a bounds dict, single-contour glyphs yield zero centroids, empty glyphs emit nothing.
  - Asserts the per-letter hint structure exposes `dot_centroids` + `glyph_bbox` and contains no `contours`/`points`/`polyline`/`referenceStrokes` keys.
- Added `tools/authoring_hints.json` as a documented stub (fonttools was absent in this environment, so real hints could not be generated). The stub carries the `_note` deprecation banner and a `_stub` field explaining it is regenerated by running the extractor once `pip install fonttools` is done.
- Commit: `573d1e2`
