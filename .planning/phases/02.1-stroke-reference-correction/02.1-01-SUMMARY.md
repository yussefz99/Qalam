# Phase 02.1 Plan 01: Schema field + stroke validator + ReferencePath identity Summary

**Adds the `StrokeSpec.type` field, a pure-Dart closed-loop/direction/dot/range/order validator that makes the Phase-2 outline bug impossible to reintroduce, and the identity `ReferencePath.resolve` that gives the guide, animation, and scorer one shared geometry.**

- **Completed:** 2026-06-01

## Accomplishments
- **D-03** — `StrokeSpec.type` (line|curve|dot) parsed as `json['type'] as String? ?? 'line'`, so all existing Phase-2 `letters.json` data still parses.
- **D-04** — `lib/core/scoring/stroke_validation.dart` (pure Dart, 243 lines) implements the five named checks: NOT-CLOSED, DIRECTION, DOT, RANGE, ORDER. Returns human-readable violation messages (empty == valid) so the same guard serves tests, the authoring tool, and a future load-time assertion.
- **Regression guard** — `test/curriculum/stroke_spec_validation_test.dart` feeds alif's original 64-point closed-outline loop (first≈last 0.22, perimeter/diagonal ≈2.3) and asserts rejection — the original bug is now permanently caught (threat T-02.1-01 mitigated).
- **S1-04** — `lib/core/scoring/reference_path.dart` exposes `ReferencePath.resolve` as a deterministic, order-preserving identity over authored centerline points; the guide/animation/scorer all resolve through it.
