<objective>
Stand up the Python `firebase-admin` tooling that migrates curriculum into Firestore and keeps
the bundled offline snapshot refreshable: a `seed_firestore.py` (letters.json + lessons.json →
`letters`/`lessons` collections + `meta/toleranceRamp` doc, idempotent by doc id) and an
`export_firestore.py` (Firestore → refreshed `assets/curriculum/letters.json` + `lessons.json`).

path. A `test_roundtrip.py` proves seed→export is lossless against the source JSON. This plan also
owns the two console/region/key human steps: create/confirm the Firestore database + permanent
region, and download the gitignored service-account key.

(Python firebase-admin seed), D-15 (refreshable export), D-07 (defaultToleranceRamp as a meta
doc), D-14 (ongoing edits via the console — documented operating policy, no custom authoring tool
this phase), D-16 (Firestore Native, Spark), and the service-account-key child-safety/secret
handling.

committed.
