Extend `CurriculumRepository` — the single curriculum seam (D-04) — to read letters, lessons,
and the tolerance ramp live from Firestore via a one-shot `.get()`, falling back to the existing
bundled-JSON loader on a cold no-network first run, running `validateReferenceStrokes` over
whichever source won, and caching the validated result in the existing kept-alive in-memory
cache. The existing async-getter API and the `.fromStrings` test seam are preserved unchanged;
a new `.withFirestore(FirebaseFirestore)` constructor lets tests inject a `fake_cloud_firestore`
instance. The read is once-at-boot into cache, so the practice/scoring path never blocks on a
network round-trip (PLAT-01 preserved).

D-02 (bundled-seed fallback, HARD/PLAT-01), D-03 (offline persistence; read never blocks the

and the read half of D-07 (ramp from meta/toleranceRamp). Backs ROADMAP SC#2 (round-trip into
the Dart models) and SC#4 (every flow still works fully offline — the boot read is cache/bundle
served, never blocking practice).

Output: the extended repository, the dev dependency for the Firestore test fake, and a new

|-----------|----------|-----------|-------------|-----------------|
| T-06.1-10 | Tampering | An invalid/malicious Firestore stroke reaching the scorer | mitigate | `validateReferenceStrokes` runs over the Firestore source too (D-05); a violation throws and is never cached (Task 2 Test 3) |
| T-06.1-11 | Denial of Service | Practice path blocking on a Firestore network round-trip | mitigate | One-shot `.get()` at boot into kept-alive cache (D-03); offline persistence serves cache; bundle fallback on cold first run (D-02) — the trace path never awaits the network |
| T-06.1-12 | Information Disclosure | Unauthorized read of curriculum content | transfer | Enforced server-side by Firestore rules (Plan 05, D-10: read requires request.auth != null); the repository assumes the anonymous identity from Plan 01 is already signed in |
</threat_model>
