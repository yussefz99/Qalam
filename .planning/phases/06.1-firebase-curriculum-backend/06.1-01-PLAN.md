## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → Firebase Auth | App requests an anonymous identity; tokens managed by Firebase |
| device → repo (git) | Generated Firebase config + (never) the service-account key cross into version control |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06.1-01 | Information Disclosure | Child PII via auth | mitigate | Anonymous sign-in only (D-09b); no login UI, no name/email field; assert via grep gate in Task 3 acceptance |
| T-06.1-02 | Elevation of Privilege | Service-account key committed to git | mitigate | `.gitignore` excludes `tools/firebase/*serviceAccount*.json` (Task 2); key handled only in Plan 03 tooling |
| T-06.1-03 | Spoofing | Spoofed app obtaining an anonymous identity | accept | App Check deferred to Phase 10 (D-10a); anonymous identity is read-only via rules (Plan 05); low blast radius, no PII |
| T-06.1-04 | Tampering | Committed google-services.json leaking project identifiers | accept | Firebase docs: google-services.json is not a secret (research Q2); contains only project identifiers, not credentials |
| T-06.1-SC | Tampering | flutter pub add / dart pub global activate installs | mitigate | All FlutterFire packages are official Firebase plugins confirmed on pub.dev (research §Package Legitimacy Audit); no [ASSUMED]/[SUS] Dart packages require a blocking checkpoint here |
