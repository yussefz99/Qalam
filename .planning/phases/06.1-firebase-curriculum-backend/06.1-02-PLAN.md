<objective>
Build the shared codec that crosses the Firestore boundary in BOTH directions and BOTH
languages: the Dart `firestore_curriculum_codec.dart` (Firestore document map тЖТ Letter/Lesson
model, and model/JSON тЖТ Firestore map) and the Python `point_codec.py` ([x,y]тЗД{x,y}). The
single highest-risk detail in this phase is the Firestore nested-array constraint тАФ
`StrokeSpec.points` is `List<List<double>>` and Firestore forbids arrays-of-arrays тАФ so every
point pair must become a `{x, y}` map on write and rebuild to `[x, y]` on read. This codec is
the spine that the seed script (Plan 03), the export script (Plan 03), and the repository read
path (Plan 04) all depend on, so it is built and unit-tested FIRST, with no Firebase project or
network needed.

Purpose: Foundation for ROADMAP SC#2 (letters/lessons round-trip losslessly with the Dart
models) and SC#3 (alif + skeleton migrate without field loss). Implements the data-mapping core
of LOCKED decisions D-06 (doc-per-letter nested), D-08 (round-trip parity is a gate), and the
point-codec required by D-13/D-15.

Output: the Dart codec, the Python codec, and round-trip parity tests covering alif (with its
strokes) and at least one skeleton letter (empty referenceStrokes, signedOff:false).
</objective>
