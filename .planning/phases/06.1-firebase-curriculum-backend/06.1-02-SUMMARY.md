# Phase 06.1 Plan 02: Firestore Curriculum Codec Summary

**The shared {x,y}<->[x,y] codec that crosses the Firestore boundary in both directions and both languages — solving the nested-array landmine once (Dart + Python), with round-trip parity proven against the real alif, a skeleton letter, and the tolerance ramp.**

## Performance

- **Duration:** ~9 min
- **Completed:** 2026-06-14
- **Tasks:** 2 (Task 1 TDD: RED -> GREEN; Task 2 auto)
- **Files modified:** 3 (3 created, 0 modified)

## Accomplishments
- **The nested-array landmine is solved once, shared, and tested in both languages (D-06).** `StrokeSpec.points` (`List<List<double>>`) now maps to `[{x,y}, ...]` on the Firestore side and rebuilds losslessly to `[[x,y], ...]` on read — Firestore's array-of-arrays prohibition is handled in a single place that the seed (Plan 03), export (Plan 03), and repo read (Plan 04) all depend on.
- **Dart codec (`lib/data/firestore_curriculum_codec.dart`)** exports `letterFromFirestore` / `letterToFirestoreMap` / `lessonFromFirestore` / `lessonToFirestoreMap` plus `encodePoints` / `decodePoints` and the meta-ramp helpers. It is **pure Dart** (no `cloud_firestore` import — acceptance criterion met) and **defers to `Letter.fromJson` / `Lesson.fromJson`** so the existing field contract (type default, nullable audio/tolerances, signedOff, mistakesStatus, defensive ramp parse) is reused verbatim, never re-implemented.
- **Round-trip parity proven (D-08, foundation for SC#2/SC#3):** alif (with its real stroke) deep-equals the bundle parse across every field; a skeleton letter survives with empty `referenceStrokes` + `signedOff:false` + `mistakesStatus:'placeholder'` intact (Pitfall 6); a lesson + the file-level `defaultToleranceRamp` survive the meta-doc round-trip (Pitfall 5).
- **Python codec (`tools/firebase/point_codec.py`)** applies the SAME transform field-for-field (`encode_points` / `decode_points` / `encode_letter` / `decode_letter`) and **self-verifies** — running it asserts `decode(encode(x))==x` over all 28 real letters, exits 0, prints OK. Stdlib-only (`json`), no firebase-admin, runs with bare `python`.
- **`meta/toleranceRamp` doc layout decided (D-07, resolves Research Open Q4):** collection `meta`, doc id `toleranceRamp`, field `ramp: [..]` — carried by `metaToleranceRampToFirestore` / `metaToleranceRampFromFirestore`.

## Decisions Made
- **Codec on `Map<String,dynamic>`, not `cloud_firestore` types** — keeps it pure-Dart unit-testable (the acceptance criterion) and reusable by the Plan-04 repo, which will adapt `DocumentSnapshot.data()` into the map before calling `letterFromFirestore`.
- **`meta/toleranceRamp` layout** (collection `meta`, doc id `toleranceRamp`, field `ramp`) — resolves Research Open Q4 / D-07; the seed (Plan 03) writes this doc and the repo (Plan 04) reads it before falling back to the hard-coded default.
- **`num`->`double` (Dart) / `float()` (Python) on decode** — a Firestore int (e.g. a `y` that serializes as `1`) round-trips to `1.0` without breaking point parsing; the Python self-check compares against a float-normalized original so int/float JSON spelling is not treated as a lossy mismatch.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Python success message used a non-ASCII character that crashed on the Windows console**
- **Found during:** Task 2 (first run of the self-check)
- **Issue:** The self-check `print` used the ring operator (`decode∘encode`); the Windows console codec (cp1255) raised `UnicodeEncodeError` and the script exited 1 — even though the round-trip logic itself succeeded over all 28 letters. The plan's verify command greps for `OK`, which never printed.
- **Fix:** Replaced the ring operator with ASCII (`decode(encode(x)) == x`) in the success message. No logic change.
- **Files modified:** `tools/firebase/point_codec.py`
- **Commit:** `2f8a1a8`
