# Phase 06.1 Plan 03: Python Firebase Seed + Export Tooling Summary

**The Python `firebase-admin` tooling that migrates the bundled curriculum into the live me-west1 Firestore and keeps the offline snapshot refreshable — a doc-id-idempotent seed, a byte-clean export, and a key-free parity gate, all reusing Plan 02's shared point codec; verified end-to-end against the real database (28 letters + 28 lessons + the tolerance ramp, lossless).**

## Accomplishments
- **Live Firestore seeded and verified (D-13, ROADMAP SC#3).** `seed_firestore.py` loaded all 28 letters (alif signed-off with its real stroke + 27 skeletons), all 28 lessons, and `meta/toleranceRamp` into the production me-west1 database. Re-running it left the collections at exactly 28 + 28 + 1 docs — idempotency proven by a live re-seed and a doc count.
- **Export refreshes the bundle losslessly (D-15, ROADMAP SC#4 export half).** `export_firestore.py` reads the live collections, decodes points back to `[x,y]`, reassembles the exact source JSON shapes (sorted by `introOrder` / `order`), and writes 2-space-indented UTF-8. After canonicalizing key order, a live export produced a **byte-clean `git diff`**, and a second export was stable (zero further diff).
- **Off-device parity gate (D-08).** `test_roundtrip.py` proves `seed encode -> export decode == source` without the SA key or a live DB (CI-safe), asserting: alif points decode to `[x,y]` pairs (not `{x,y}` maps), alif `signedOff==true`, every empty-stroke letter keeps `signedOff==false`, and the ramp survives. An injected mismatch was verified to exit non-zero with a field-level diff.
- **Shared codec reused (D-06).** Both scripts import `encode_letter`/`decode_letter` from Plan 02's `point_codec.py`, so the `{x,y}<->[x,y]` transform is defined once and agrees with the Dart read path.
- **Secret invariant held (threat T-06.1-07).** The admin-SDK service-account key was used only via `GOOGLE_APPLICATION_CREDENTIALS`, never staged, never printed; `git check-ignore` confirms it is excluded at every commit.
- **README captures the D-14 operating policy:** ongoing curriculum edits happen in the Firebase Console (doc-per-letter is console-friendly); there is no custom authoring tool this phase; `export_firestore.py` is how console edits flow back into the offline bundle.

## Decisions Made
- **Canonical nested-key order in the export.** Firestore does not preserve map key order, so without normalization the export's `git diff` drifted purely on key ordering of `forms`, `commonMistakes`, `tolerances`, `audio`, lesson `title`/`items`/`unlock`. The export now re-orders every nested object to one canonical order, and `letters.json` was canonicalized to match. Verified `json.loads(exported) == json.loads(original)` — zero semantic change.
- **Test reuses the export reassembly.** `test_roundtrip.py` imports `_normalize_letter`/`_normalize_lesson` from the export so the off-device gate exercises the exact code path the live export runs — the test cannot pass while the live export would diff.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Export diffed on nested-object key order (Firestore loses map ordering)**
- **Found during:** Task 3 (first live export)
- **Issue:** The first live export produced a non-trivial `git diff` against the source JSON. Inspection showed every changed line was a key-order swap inside nested objects (`forms`, `commonMistakes[]`, `tolerances`/`audio`, lesson `title`/`items`/`unlock`) — `json.loads` of the two files was value-equal. Firestore does not preserve map key order, so the export emitted a different (but valid) ordering, which failed the "live diff clean" success criterion.
- **Fix:** Extended `export_firestore.py` with canonical key-order tables for every nested object and a `_normalize_lesson` helper; re-ordered the source `letters.json` to the same canonical order (a one-time, value-preserving canonicalization). A second live export against me-west1 is now byte-stable (zero diff). The off-device test was updated to import the same normalizers.
- **Files modified:** `tools/firebase/export_firestore.py`, `assets/curriculum/letters.json`, `tools/firebase/test_roundtrip.py`
- **Commit:** `55f14ec`
