## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| device → Google ML Kit model CDN | One-time model fetch over the network (the only external fetch in this phase) |
| child strokes (in-memory) → ML Kit Ink | Untrusted points converted to an Ink for on-device recognition; never transmitted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | ML Kit model download | mitigate | Fetch only via `DigitalInkRecognizerModelManager` from Google's own infrastructure; no custom model URLs; verify `isModelDownloaded` before any `recognize` (Pitfall 5). |
| T-04-06 | Availability/DoS | model not present on first run / offline | mitigate | D-05 background fetch + best-effort try/catch → calm "getting ready" state, never an error or hard-block; fully offline after the one-time fetch. |
| T-04-07 | Information Disclosure | child strokes → ML Kit Ink | mitigate | Recognition is on-device only; strokes are converted in-memory and never transmitted or logged (T-01-05). No PII in the Ink. |
| T-04-SC | Tampering | pub.dev install (`google_mlkit_digital_ink_recognition`) | mitigate | Owner-validated + STACK.md-prescribed + official Google `google_ml_kit_flutter` family (Package Legitimacy Audit: Approved). slopcheck targets npm/PyPI/crates, N/A for pub.dev; no human-verify checkpoint required for this single already-decided package. |
</threat_model>
