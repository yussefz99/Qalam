<objective>
Wire the full thin loop end-to-end for alif (D-01): a minimal Start entry point (D-02)
into a Watch → Trace → Celebrate state machine. The `practiceSessionController` (Riverpod
Notifier, family by lessonId) loads alif via `getLesson('lesson_01')`, drives the three
phases, counts clean reps, and on the 3rd clean rep (`cleanRepsToAdvance: 3`, D-07) calls
`progressRepository.recordMastery(...)` (Plan 02) and transitions to a dignified
celebration (D-08). The screen feeds completed strokes from the `StrokeCanvas` (Plan 03)
into `scoreStroke` (Plan 01); a miss highlights the failing stroke + shows the named fix
from `commonMistakes[].feedback` in coral with unlimited gentle retries (D-04/D-05); a
clean pass advances the rep. All anti-gamification chrome from the mockups is omitted
(D-03/D-06/D-08/D-12, PLAT-03): no running star total, no weekly tally, no "Play sound", no
"Mark correct", no "See journey", no confetti — exactly ONE quiet mastery star.

Purpose: This is the phase's payoff — SC1–SC4 become true together. It assembles the
already-built scorer, persistence, capture, and animation into the real child experience.

Output: l10n strings, `practice_providers.dart`, `feedback_panel.dart`,
`mastery_celebration.dart`, `practice_screen.dart`, the router entry, and the session /
anti-gamification / celebration tests.
</objective>
