<objective>
Build the pure-Dart geometric stroke scorer — the deepest-risk, highest-value artifact of
this phase — TDD-first. The scorer judges a single captured stroke against alif's authored
reference stroke for direction, vertical extent, and straightness, and maps any failure to
exactly one of alif's three named `commonMistakes` (`too_short` / `wrong_direction` /
`too_curved`). It is a structural sibling of `lib/core/scoring/stroke_validation.dart`:
pure Dart, no Flutter import, named checks whose names equal the authored
`commonMistakes[].check` strings, all tuned thresholds in ONE documented block (lenient
first-cut per D-15; Phase 4 calibrates).

Purpose: This is the "feedback" half of S1-05 — the thing that makes the child's trace
mean something. Built pure-Dart so the owner's mother can validate the pedagogy via tests
in Phase 4 without a widget harness.

Output: `scoring_models.dart`, `stroke_resampler.dart`, `geometric_stroke_scorer.dart`,
their unit tests, and shared synthetic stroke fixtures.
</objective>
