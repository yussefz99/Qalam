> Visual and interaction contract for the Watch → Trace → Celebrate core loop (alif).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Source of truth for visuals: `docs/design/kit/` (Qalam Design System) + the in-code
> theme tokens (`lib/theme/`). Build the three screens **pixel-faithfully to the design**

## Scope Note (read first)

This phase is **Flutter / Android / RTL** — NOT a React/Next/Vite web app. The shadcn
initialization gate does not apply (`Tool: none`). The design system is already
**fully codified** in `lib/theme/` (one-way translation from `colors_and_type.css`):
`QalamColors`, `QalamSpace`, `QalamTargets`, `QalamRadii`, `QalamShadows`, `QalamMotion`,
`QalamInk`, `QalamFonts`, `QalamFontSizes`, `QalamTextStyles`. **Widgets read these
tokens — never raw hex, never magic numbers, never inline fonts.** No new tokens are
introduced by this phase.

The mockups depict the letter **baa**; Phase 3 ships **alif** (the one signed-off
letter). Reuse the exact layout, swap the glyph and copy to alif.

## Spacing Scale

Declared values (4px base — `QalamSpace`; all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| space1 | 4px | Hairline gaps, inline padding |
| space2 | 8px | Compact element spacing |
| space3 | 12px | Tight stacks |
| space4 | 16px | Default gap; **minimum gap between touch targets** (kids-UX floor) |
| space5 | 20px | Comfortable inner padding |
| space6 | 24px | Card padding / section padding |
| space8 | 32px | Group gaps |
| space10 | 40px | Layout gaps |
| space12 | 48px | Major section breaks |
| space16 | 64px | Page-level spacing / canvas margins |
| space20 | 80px | Hero spacing |
| space24 | 96px | Full-screen celebration breathing room |

**Touch-target floors (`QalamTargets`, kids-UX — binding):** interactive elements
≥ **64px** (`targetMin`); comfortable 72px (`targetComfy`); hero CTAs 96px
(`targetLarge`). Minimum 16px apart. The "I'll try", "Watch again", "Replay",
"Try again", "Next", and "Back home" buttons are hero/comfy targets.

Exceptions: the numbered **start-dot** marker and the animated **pen-tip** are
non-interactive visual indicators (not touch targets) and are sized for visibility,
not for the 64px floor. One task per screen (kids-UX).

## Color

60 / 30 / 10 split using `QalamColors` (translated from `colors_and_type.css`):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAF6EE` `QalamColors.bg` (parchment) | App background, the trace canvas surface — warm, **never** stark white |
| Secondary (30%) | `#EAF4F4` `QalamColors.surface` (soft aqua) + `#FFFFFF` `surfaceRaised` (sparingly) | Tip card, listen-area-shaped panels, raised canvas card, nav rail |
| Accent (10%) | `#168A8F` `QalamColors.primary` (ink-teal) | Primary CTAs ("I'll try", "Next", "Back home"), the demo/ink stroke color (`inkStroke` = deep-ink `#0E5B5F`), active nav, headings on celebration badge |
| Reward (gold) | `#F2A60C` `QalamColors.reward` | **REWARDS ONLY** — the numbered **start-dot**, the **single mastery star**, the mastered-alif glow on celebration. **Never** a button, never a heading, never body. If gold appears, something good just happened. |
| Success | `#3FB984` `QalamColors.success` (leaf) | A clean-stroke pass confirmation (subtle), "Stroke X of N" advance tick |
| Soft-warn | `#FF8A6B` `QalamColors.warnSoft` (coral) | The **only** "error" treatment: highlight the failing stroke + frame the named fix warmly. **No red, no red X** — coral + soft wiggle, "let's try that again" energy |

Accent reserved for: **primary action buttons, the live/demo ink stroke, the active
nav-rail item, and the "MASTERED" badge.** NOT for all interactive elements
(secondary actions like "Watch again" / "Replay" / "Try again" use outlined/ghost
treatment on parchment, not filled teal).

**Gold is reserved for** (rewards only, exhaustive list for this phase): the numbered
**start-dot** marker, the **single mastery star** that settles in on celebration, and
the **mastered-alif glyph + soft glow** on the celebration screen. Nothing else may use
gold.

**No destructive actions in this phase.** There is no delete, no account, no
irreversible action. The closest thing is "Clear ink on retry," which is automatic,
silent, non-destructive (in-memory stroke discarded), and needs no confirmation. There
is **no fail state, no try-counter, no pressure** (D-05).

## Copywriting Contract

All user-facing copy lives in `AppLocalizations` (gen-l10n) — never hardcoded. Voice:
warm, calm, specific; second person to the child; Title Case on buttons/headings; the
mascot Qalam speaks in first person ("Watch me write it."). **Feedback always names the
exact fix — never a generic "Oops, try again!"** (PLAT-03, Pitfall 7).

### Screen labels & headings

| Element | Copy |
|---------|------|
| Watch step eyebrow | `WATCH · STROKE ORDER` |
| Watch step heading | `Watch me write alif.` |
| Watch tip card | `Start at the gold dot. Follow the line down.` (heading "Tip", "gold dot" emphasized in gold text) |
| Trace step eyebrow | `YOUR TURN · TRACE` |
| Trace step heading | `Now you trace alif.` |
| Trace progress | `Stroke 1 of 1` (alif is one stroke; the model supports `Stroke X of N`) |
| Rep progress (during session) | `1 of 3` → `2 of 3` → `3 of 3` clean reps (quiet, neutral — a progress indicator, NOT a score) |
| Celebration eyebrow | `MASTERED` |
| Celebration heading | `You learned alif.` |
| Celebration praise (Arabic island) | `أحسنت` (RTL, Noto Naskh, paired beneath/with the English line) |
| Celebration badge | `MASTERED` (teal badge with one gold star glyph) |

### Buttons (Title Case)

| Element | Copy |
|---------|------|
| Primary CTA — Watch step | `I'll Try` (advances Watch → Trace) |
| Secondary — Watch step | `Watch Again` (replays the demo; D-10) |
| Secondary — Trace step | `Replay` (replays the demo over the guide) |
| Secondary — Trace miss | `Try Again` (retry the SAME stroke after a fix; unlimited, no counter) |
| Primary CTA — Trace pass (pre-mastery) | `Next` (advance to the next clean rep) |
| Primary CTA — Celebration | `Back Home` |

### Empty / first-entry state

| Element | Copy |
|---------|------|
| Entry heading (minimal Start entry point, D-02) | `Let's learn alif.` |
| Entry body + next step | `Watch how it's written, then trace it yourself.` (single "Start" CTA → Watch step) |

> There is no "no data" empty state inside the loop — the child always has alif to
> trace. The "empty" state is simply the entry point before the loop begins.

### Feedback / "error" states (named fixes — straight from alif's authored `commonMistakes`)

Source: `assets/curriculum/letters.json` alif `commonMistakes[].feedback` (already
authored in the tutor's warm, specific voice). The scorer emits a `MistakeId`; the
presentation layer surfaces the matching string via l10n. **Frame in coral, never red;
highlight the failing stroke; let them retry.**

| Mistake id / check | Copy (named fix) |
|--------------------|------------------|
| `too_short` / `strokeLengthBelowThreshold` | `Your alif needs to be taller — draw it from the top all the way down.` |
| `wrong_direction` / `strokeDirectionInverted` | `Start your alif at the top and come down — not from the bottom up.` |
| `too_curved` / `strokeCurvatureExceedsThreshold` | `Alif is a straight line — try to keep it as straight as you can.` |
| Fallback (no matching named mistake) | `Something looks off — try again, slower this time.` (specific-ish, calm — **never** "Oops, try again!") |

> Verify these strings against the live `letters.json` at plan/implement time; the
> authored feedback is the contract between curriculum data and code. The scorer's
> predicate names MUST match the `check` values above.

### Destructive confirmation

`none` — this phase has no destructive actions. Ink-clear on retry is automatic,
silent, and non-destructive.

## Interaction & Motion Contract

| Moment | Behavior | Tokens / Source |
|--------|----------|-----------------|
| Enter Watch step | Demo **auto-plays once**, then offers Replay / Watch Again (D-10) | `AnimationController`; `QalamMotion.durSlow` for the pen draw pacing |
| Stroke-order demo | Animated pen-tip traces the **resolved reference path** from the numbered gold start-dot, correct direction, progressive dotted draw via `PathMetric.extractPath` (D-11, S1-04 one source of truth) | gold start-dot = `QalamColors.reward`; ink = `QalamColors.inkStroke`; `QalamMotion.easeOutQuart` |
| Mascot | Qalam stands beside the canvas as the **tutor persona** (write/idle states), demonstrating — pedagogical, not a game character | brand mascot SVG (`docs/design/kit/.../mascot/`); placeholder until illustrated |
| Tracing | Live ink renders smoothly under the stylus/finger; stylus-only in prod, debug-finger flag in dev (D-13/D-14); palm rejection is free (touch filtered in prod) | `QalamInk.strokeWidth = 6`, `inkStroke` deep-ink; `Listener` capture (not GestureDetector) |
| Stylus-up → feedback | Feedback appears **< ~300 ms**, fully offline (S1-05) | on-device geometric scorer; runs on main isolate |
| Miss | Failing stroke highlighted in **coral**, named-fix line shown in the tutor's voice, soft wiggle, ink clears, retry the SAME stroke. **No fail state, no counter, no pressure** (D-05) | `QalamColors.warnSoft`; `QalamMotion.easeSoftBack` for a gentle wiggle |
| Clean pass (pre-mastery) | Quiet confirmation, advance rep counter (1/3 → 2/3 → 3/3), clear for next rep | `QalamColors.success`, subtle; `QalamMotion.durBase` |
| 3rd clean rep → Celebrate | Transition to a **calm full-screen** moment: mascot (cheer state) + mastered alif (gold glyph + soft glow) + **ONE** gold star settling in + "You learned alif. أحسنت." → Back Home. **No confetti, no sound blast, no running counter** (D-08) | `QalamMotion.durCheer` (700ms) for the single-star settle; gentle, never slapstick |
| Mastery persistence | Mastery + clean-rep count written to Drift; survives restart (D-09). Stroke points stay **in-memory only**, never persisted/logged (T-01-05) | `ProgressRepository` → Drift |

RTL note: app chrome stays **LTR** (English). Arabic appears only as RTL content
islands (the alif glyph, "أحسنت"). Mirror only Arabic content blocks, not the whole app.
