<objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Flutter client → Cloud Run /coach | Untrusted child device crosses the network; must present a valid Firebase ID token + App Check token |
| Cloud Run app → Secret Manager | Provider keys injected as env vars by the runtime SA; never in image/client |
| Request body → graph | The non-PII FACTS payload; only the 6 whitelisted fields may enter (extra=forbid) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Spoofing | POST /coach caller | mitigate | `verify_caller` verifies the Firebase ID token via firebase-admin; no token → 401 before the graph (Task 1) |
| T-14-02 | Tampering/Replay | client→server token | mitigate | App Check limited-use token verified per request (`X-Firebase-AppCheck`); replay-protected (Task 1) |
| T-14-03 | Information Disclosure | request body | mitigate | `TutorFactsIn` extra=forbid → any leaked stroke/PII field is a 422 (Task 2); GROUND-02 build-failing test lands in Plan 04 |
| T-14-04 | Information Disclosure | provider API keys | mitigate | Keys in Secret Manager → env vars; grep gate asserts no key string in `server/`; image excludes .env (Task 1) |
| T-14-05 | Denial of Service | slow/hung model call | mitigate | `asyncio.wait_for` + Cloud Run `--timeout`; timeout returns fast → client degrades to AuthoredFallback (Task 2 / G5) |
| T-14-06 | Elevation | agent emits a verdict/5th action | mitigate | `tool_choice="any"` pins the closed 4-tool set; `advance`-on-fail rewritten to grounded `say` (Task 2 / G2,G3) |
| T-14-SC | Tampering | pip installs (langgraph/langchain/firebase-admin) | mitigate | RESEARCH Package Legitimacy Audit + pinned versions; all are GA, widely-used PyPI packages (see verification note below) |
</threat_model>
