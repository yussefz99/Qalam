<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| child (untrusted) → PIN verify | A child guesses PINs; PBKDF2 + persisted cooldown is the defense |
| pin material → AppSettings (app-private Drift) | Hash/salt at rest on-device; never leaves the device, never network |
| pub.dev → build (supply chain) | One new dependency (`crypto`) crosses the build boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | PIN at rest (parentPinHash) | mitigate | Salted PBKDF2-HMAC-SHA256 (≥100k iters), Random.secure() per-install salt; never store plaintext; never log pin/salt/hash (Task 2 + grep gate). |
| T-09-02 | Tampering / Elevation | brute-force cooldown | mitigate | failCount + lockUntil PERSISTED in AppSettings, not in-memory — survives force-quit (the single most important correctness point; verified by the restart test). |
| T-09-06 | Information Disclosure | timing side-channel on verify | mitigate | Constant-time XOR-accumulate compare; no early-out on first mismatch. |
| T-09-08 | Information Disclosure | reversible PIN storage | mitigate | One-way hash only; flutter_secure_storage explicitly NOT used (a one-way hash needs no recovery — research rejects it). |
| T-09-09 | Tampering | dashboard write-back | mitigate | Only read accessors added (allMastered/allInProgress); no write/edit/delete accessor wired to the parent surface (read-only hard constraint). |
| T-09-SC | Tampering | `flutter pub add crypto` | mitigate | Blocking human legitimacy checkpoint (Task 1) before install; crypto = dart.dev first-party, verified on pub.dev (09-RESEARCH audit). Not auto-approvable. |

**Threat-accepted (documented):** A 4-digit PIN + PBKDF2 (not Argon2id, not Keystore-backed) is intentionally proportionate to a LOW-VALUE local asset whose realistic adversary is a curious child, not a forensic device-extraction attacker. A root/physical attacker can read the unencrypted Drift DB regardless — this PIN gate is a CHILD BARRIER, not a device-security boundary (CLAUDE.md child-data-minimalism framing; 09-RESEARCH A1, ratified at security review).
</threat_model>
