# Phase 9 — UI Design Contract

> Visual and interaction contract for the Parent Dashboard (S1-11).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Source of truth for visuals: `docs/design/kit/` (Qalam Design System) + the in-code
> theme tokens (`lib/theme/`). This phase adds **one new top-level route** (`/parent`)
> rendering up to three states (PIN-create, PIN-enter, dashboard) and **no new design
> tokens** — every value below maps to an existing `Qalam*` token.

---

## Scope Note (read first)

This phase is **Flutter / Android / RTL** — NOT a React/Next/Vite web app. The shadcn
initialization gate does not apply (`Tool: none`). The design system is already fully
codified in `lib/theme/` (one-way translation from `colors_and_type.css`):
`QalamColors`, `QalamSpace`, `QalamTargets`, `QalamRadii`, `QalamShadows`,
`QalamMotion`, `QalamFonts`, `QalamFontSizes`, `QalamTextStyles`.
**Widgets read these tokens — never raw hex, never magic numbers, never inline fonts.**

This is the app's **only adult space** (D-07). It must read as calm, plain, and
distinct from the child's playful Home: no mascot, no celebration motion, no gold, no
prepared-desk entrance, no rounded "sticker" hype. The presentational model is the
existing `settings_screen.dart` (a centered `ConstrainedBox(maxWidth: 640)` column on
`QalamColors.bg`), NOT the `home_screen.dart` greeting/today-card layout.

### Surfaces this phase adds

| Surface | File (planner's proposed paths) | What it renders |
|---------|--------------------------------|-----------------|
| Home "Parent" nav item (unlock) | `lib/screens/home_screen.dart` `_NavItem` | Drop `isLocked`/`sublabel`/`comingSoon`; swap `lock.svg` → a non-lock parent icon; wire `onTap: () => context.go('/parent')` (D-06) |
| PIN create | `lib/features/parent/parent_pin_gate.dart` | Enter + confirm a 4-digit PIN with the honest no-recovery line (D-01/D-02) |
| PIN enter | `lib/features/parent/parent_pin_gate.dart` | Obscured 4-digit entry; wrong-PIN feedback; calm cooldown state (D-08) |
| Dashboard (read-only) | `lib/features/parent/parent_dashboard_screen.dart` | Summary line + scrollable per-letter list + empty state (D-03/D-04/D-05) |

### Mandatory omissions (anti-gamification — PLAT-03, binding)

| Forbidden | Rule |
|-----------|------|
| Running point total, weekly tally, "+N", streak, badge, leaderboard | OMIT everywhere |
| A 0–100 score, grade, or percentage UI | D-05 — there is NO numeric score; do NOT invent one |
| Gold/reward token (`QalamColors.reward`) anywhere on these screens | This is an adult info view, not a reward moment — gold means a mastery happened, which never happens *here* |
| Mascot, celebration motion, child-game chrome | D-07 — distinct adult space |
| Any edit / delete / reset affordance on the dashboard | Read-only is a hard constraint (Discretion) |
| Eastern-Arabic digits (٠١٢) | Western numerals everywhere, LTR-isolated inside Arabic islands (Phase-1 decision; `ArabicText` handles it) |
| The entered PIN echoed, autofilled, or logged | `obscureText:true`, numeric keyboard, never `print`/`debugPrint` PIN material (Pitfall 4 / no-log convention) |

## Spacing Scale

Declared values (4px base — `QalamSpace`; all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| space1 | 4px | Hairline gaps (row metadata inline) |
| space2 | 8px | Compact stacks (status label ↔ rep count) |
| space3 | 12px | Between list rows (matches `settings_screen` `_PlaceholderRow` margin) |
| space4 | 16px | Default gap; **minimum gap between touch targets**; PIN-pad inter-key gap |
| space5 | 20px | List-row horizontal padding (matches `_PlaceholderRow`) |
| space6 | 24px | Summary line ↔ list gap; row inner vertical rhythm |
| space8 | 32px | Screen padding (matches `settings_screen` `EdgeInsets.all(space8)`); group gaps |
| space12 | 48px | Major vertical break (PIN title block ↔ keypad; empty-state breathing room) |

**Touch-target floors (`QalamTargets` — binding):** every interactive element ≥ **64px**
(`targetMin`). The **PIN keypad keys ≥ 72px** (`targetComfy`) — this is a child-proof
adult keypad on a tablet, so larger keys reduce mis-taps; ≥ 16px (`space4`) between keys.
The "Done"/back control ≥ `targetComfy`. The primary PIN-confirm button ≥ `targetComfy`.

**Layout container:** centered `ConstrainedBox(maxWidth: 640)` column on `QalamColors.bg`,
mirroring `settings_screen.dart`. The dashboard list scrolls inside this width; the PIN
screens center their content vertically as well as horizontally.

Exceptions: the per-letter Arabic glyph and the `check-complete` status glyph are
**non-interactive** visual indicators, sized for legibility (≈ `space8`–`space10`), not
for the 64px floor. The 4 PIN-state dots are non-interactive indicators (≈ `space3` each).

## Typography

Two **script-isolated** scales, as established in 03-/06-UI-SPEC: an English scale
(LTR chrome — everything here) and an Arabic scale (RTL islands — only the per-letter
labels). This phase introduces **no new type roles** — every element maps to an existing
`QalamTextStyles` role.

### English type scale — exactly 4 roles for this phase

| Role | Size | Weight | Line Height | Font | Used for (this phase) |
|------|------|--------|-------------|------|----------------------|
| Heading | 28px | 500 | 1.3 | Fredoka | "Parent Area" screen title; "Create a PIN" / "Enter your PIN" prompts; the summary line "N of 28 letters mastered" |
| Button | 24px | 500 | 1.0 | Fredoka | "Continue" / "Confirm" / "Done" button labels |
| Body | 18px | 400 | 1.5 | Nunito | The honest no-recovery line; cooldown line; wrong-PIN line; per-letter row metadata (rep count, mastered date); empty-state body |
| Label | 16px | 600 | 1.3 | Nunito | Per-row status label ("Mastered" / "In progress"); section eyebrow if any; the "N of 28" denominator is part of the Heading-role summary |

> **4-size scale for this phase: 28 → 24 → 18 → 16.** Two weights per family:
> Fredoka 500, Nunito 400 (body) / 600 (label). No 42px Display role on these screens —
> this is a quiet adult view, so the largest role is the 28px Heading (deliberately
> smaller/calmer than the child Home's 42px greeting).

### Arabic type scale — RTL islands only (per-letter labels)

Never bold, never italic, never letterSpacing (`letterSpacing: 0` explicit — Pitfall 2).
Render via `ArabicText` (the only RTL island; chrome stays LTR — D-05).

| Role | Size | Weight | Line Height | Font | Used for (this phase) |
|------|------|--------|-------------|------|----------------------|
| Arabic body | 26px | 400 | 1.7 | Noto Naskh | The per-letter glyph label in each dashboard row (e.g. ا, ب) via `ArabicText(letter.char)` |

Western numerals everywhere (rep counts, dates, "N of 28") — LTR-isolated automatically
inside any Arabic island by `ArabicText`; in the English chrome they render as plain
Nunito with `tnum` where columnar alignment helps (rep counts).

## Color

60 / 30 / 10 split using `QalamColors` — a deliberately quiet, accent-light palette
fitting an adult info view:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAF6EE` `QalamColors.bg` (parchment) | All three states' background — never stark white |
| Secondary (30%) | `#EAF4F4` `QalamColors.surface` (soft aqua) | Per-letter list-row surface (rounded card, like `_PlaceholderRow`); PIN-pad key faces; summary-line container if used |
| Accent (10%) | `#168A8F` `QalamColors.primary` (ink-teal) | See reserved-for list below |
| Success | `#3FB984` `QalamColors.success` (leaf) | The `check-complete` "Mastered" marker glyph + its "Mastered" status label tint |
| Soft-warn | `#FF8A6B` `QalamColors.warnSoft` (coral) | Wrong-PIN feedback text + the 4-dot indicator on a wrong attempt — the only "error" color; **never red, never a red X** |
| Muted text | `#5C6B70` `QalamColors.fgMuted` (slate) | "In progress" status label; rep-count + date metadata; cooldown countdown line |

**Accent (teal) reserved for** (exhaustive): the primary PIN button ("Continue" /
"Confirm") fill; the "Done" control; the active/filled PIN-dot indicators; the unlocked
"Parent" nav-rail item when active. NOT for list rows, NOT for status markers, NOT for
the keypad digit keys (those are `surface` faces with `fg` digits).

**Reward (gold) — FORBIDDEN on these screens.** Although the dashboard reports mastery,
this is an adult information view, not a celebration. Mastered letters are marked with
the **leaf `check-complete` glyph**, never a gold star. Gold appears nowhere in Phase 9.

**Status encoding (D-03/D-05):**
- **Mastered** → `check-complete.svg` glyph tinted `QalamColors.success` (leaf) + a
  16px/600 "Mastered" label in `success`; trailing metadata: clean-reps count +
  mastered date (Body/18px, `fgMuted`).
- **In progress** → no glyph (or a faint neutral dot in `border`); "In progress" label
  16px/600 in `fgMuted`; trailing metadata: clean-reps count only, no date.

**Destructive:** none. The only destructive *concept* — "forgot PIN = clear app data" —
is **stated as honest informational copy at PIN creation**, not an in-app destructive
button. Phase 9 ships no clear-data control (the OS settings handle app-data clearing).

## Copywriting Contract

All user-facing copy lives in `AppLocalizations` (gen-l10n) — never hardcoded; English,
LTR chrome. Voice: calm, plain, adult-to-adult here (this is the one screen NOT addressed
to the child) — but still warm and never alarming. Title Case on buttons.

> **Owner sign-off required (Open Q2):** the exact honest no-recovery line and the
> cooldown line are drafted below in brand voice but the **owner owns final wording**
> (CLAUDE.md). These are the planner's proposed ARB strings.

### PIN — create (first access, D-01/D-02)

| Element | Copy | Notes |
|---------|------|-------|
| Screen title | `Parent Area` | NEW key `parentTitle`; Heading/28px |
| Prompt | `Create a PIN` | NEW key `parentPinCreatePrompt`; Heading/28px |
| Step 1 helper | `Choose a 4-digit PIN to protect this area.` | NEW key `parentPinCreateHelp`; Body/18px |
| Step 2 prompt | `Re-enter your PIN to confirm.` | NEW key `parentPinConfirmPrompt`; Body/18px |
| Mismatch (step 2) | `Those didn't match. Try once more.` | NEW key `parentPinMismatch`; Body/18px, `warnSoft` — calm, no blame, returns to step 1 |
| Honest no-recovery line | `There's no way to recover a forgotten PIN. The only reset is clearing the app's data — which also erases your child's progress.` | NEW key `parentPinNoRecovery`; Body/18px, `fgMuted`. Shown beneath the create field. Honest, plain, non-alarming (D-02). |
| Primary CTA (step 1) | `Continue` | NEW key `commonContinue` (or reuse if present); enabled only at 4 digits |
| Primary CTA (step 2) | `Confirm` | NEW key `parentPinConfirm`; enabled only at 4 digits |

### PIN — enter (every subsequent entry, D-07)

| Element | Copy | Notes |
|---------|------|-------|
| Screen title | `Parent Area` | reuse `parentTitle` |
| Prompt | `Enter your PIN` | NEW key `parentPinEnterPrompt`; Heading/28px |
| Wrong PIN (generic, no oracle) | `Incorrect PIN. Try again.` | NEW key `parentPinWrong`; Body/18px, `warnSoft`. **Generic** — never reveals which digit (V7). 4-dot indicator clears |
| Cooldown (after ~5 wrong, D-08) | `Too many tries. You can try again in {seconds}s.` | NEW key `parentPinCooldown` with `{seconds}` placeholder; Body/18px, `fgMuted`. Live countdown. Calm, not an alarm — keypad disabled, no red, no lock-out language |
| No CTA on this screen | Verifying happens on the 4th digit (auto-submit) OR a "Continue" button — see Interaction Contract | — |

### Dashboard (read-only, D-03/D-04/D-05)

| Element | Copy | Notes |
|---------|------|-------|
| Screen title | `Parent Area` | reuse `parentTitle` |
| Summary line | `{mastered} of {total} letters mastered` | NEW key `parentSummary` with `{mastered}`/`{total}` placeholders; Heading/28px. `{total}` = `curriculumRepository.getLetters().length` (Open Q1 — default full 28). Plain information, NOT a score treatment, never gold |
| Row — mastered | `Mastered · {reps} clean reps · {date}` | NEW key `parentRowMastered`; "Mastered" is the Label (leaf), the rest Body/`fgMuted`. Date format = device-locale short date (executor discretion; Western numerals) |
| Row — in progress | `In progress · {reps} clean reps` | NEW key `parentRowInProgress`; Label (`fgMuted`) + Body/`fgMuted`. No date |
| Empty-state heading | `No lessons completed yet.` | NEW key `parentEmptyTitle`; Heading/28px. Calm — never an error or empty void (D-04) |
| Empty-state body | `When your child finishes a letter, it will appear here.` | NEW key `parentEmptyBody`; Body/18px, `fgMuted` |
| Exit control | `Done` | NEW key `commonDone` (or reuse); Button/24px. Returns to child Home and **relocks** (D-07) |

### Loading / error (dashboard provider states)

| State | Treatment |
|-------|-----------|
| Loading | Quiet — render the title + an empty body area with no spinner chrome and no skeleton shimmer (mirrors the `_GreetingHeader`/today-card degradation pattern). Progress data is local and near-instant |
| Error | If progress fails to load, show the empty-state copy rather than a raw error (the parent area must never crash to a stack trace). Reuse existing `databaseError` copy only for a true shell-level DB failure. Never expose PIN material in any error |

### Destructive confirmation

`none` — Phase 9 ships no destructive action. The no-recovery consequence is
disclosed as honest copy at PIN creation (see `parentPinNoRecovery`), not a button.

## Interaction & Motion Contract

| Moment | Behavior | Tokens / Source |
|--------|----------|-----------------|
| Home "Parent" tap (D-06) | `context.go('/parent')`. The nav item is unlocked (no `isLocked`, no "Coming soon" sublabel, no lock glyph) | existing `_NavItem` |
| Route gate (D-06/D-07) | A **synchronous** go_router redirect guards `/parent` behind a `parentUnlocked` flag (mirror the onboarding gate). The `/parent` widget itself chooses PIN-create vs PIN-enter vs dashboard from `isPinSet` + `unlocked` (async reads inside the screen, **never** inside `redirect` — repo pitfall) | `app_router.dart` onboarding-gate pattern |
| PIN entry field | Obscured numeric input: `obscureText:true`, `keyboardType: TextInputType.number`, `enableSuggestions:false`, `autocorrect:false`, `maxLength:4`. A 4-dot indicator reflects entered-digit count; digits never echoed. **PIN never logged** | Pitfall 4 / V5 |
| PIN keypad | An on-screen numeric keypad (0–9 + backspace) with `targetComfy` (72px) keys on `surface` faces, `fg` digits, ≥16px apart — child-proof tablet sizing. (A plain obscured TextField is the research-acceptable MVP minimum; the on-screen keypad is the prescribed presentation for a tablet adult-gate — executor may use the device numeric keyboard if a keypad widget is out of scope, but keys/targets must still meet the floor) | `QalamTargets.targetComfy`, `QalamSpace.space4` |
| Submit | On the 4th digit, verify (auto-submit) for *enter*; *create* requires explicit "Continue"/"Confirm" so the parent can review/correct before locking it in | — |
| Wrong PIN (D-08) | Dots clear; show `parentPinWrong` in `warnSoft`; a **single gentle horizontal wiggle** of the dot row (`easeInOut`, `durFast` 140ms, amplitude ≤ 8px) — never a red flash, never a red X (brand: "Let's try that again", soft) | `QalamMotion.easeInOut`, `durFast`; `warnSoft` |
| Cooldown (D-08) | After ~5 wrong attempts the keypad/field disables and shows `parentPinCooldown` with a live `{seconds}` countdown (1s tick). Calm `fgMuted` text, no alarm color, no permanent lockout. **The counter + lock-until are persisted in Drift, not in-memory** (a child can force-quit to reset an in-memory counter — the single most important correctness point; this is data-layer, surfaced here only as the disabled state) | research Pattern 2 |
| Verified → dashboard | Set `parentUnlocked = true`; render the dashboard. No celebratory transition — a plain fade-in at most (`durBase` 220ms, `easeOutQuart`) or instant. **No prepared-desk entrance, no mascot** (this is the adult space) | `QalamMotion.durBase` (optional) |
| Dashboard scroll | The per-letter list scrolls inside the `maxWidth:640` column; summary line stays at top (may pin or scroll with content — executor discretion) | — |
| "Done" / back (D-07) | Always sets `parentUnlocked = false` **before** `context.go('/')`, and resets on dispose, so the next entry re-prompts the PIN (per-entry, no session unlock). The Android system-back must follow the same relock path | Pitfall 6 |

**RTL note:** app chrome stays **LTR** (English) — titles, prompts, buttons, status
labels, the summary line. Arabic appears **only** as the per-letter glyph label, via
`ArabicText` (an RTL island). Mirror only that island, never the whole screen (D-05).

**Reduced motion:** if `MediaQuery.disableAnimations` is true, skip the wrong-PIN wiggle
and any dashboard fade — render settled immediately (consistent with Phase 6).
