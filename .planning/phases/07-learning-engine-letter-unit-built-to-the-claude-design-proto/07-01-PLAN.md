Build the Curriculum Schema v2 data spine in Dart and extend CurriculumRepository to load it.

seed (structure only; `signedOff:false`, content TBD).

models — this plan defines the contracts they consume, interface-first, so executors never guess.

and round-trip tests proving all 19 configs deserialize losslessly (CUR-01 engine).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Firestore → app (curriculum read) | exercises/words/units docs are untrusted-until-validated content read into the app |
| bundled asset → app | bundled-seed JSON is trusted (ships in the APK) but must not silently mask a bad Firestore payload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering | curriculum read path | mitigate | exercises/words/units are read-only to the client; writes denied by Firestore rules (extended in Plan 07-02). This plan never writes Firestore. |
| T-07-01-02 | Information disclosure | model files | mitigate | Schema v2 models hold curriculum content only — no child PII, no stroke data, no progress. Models stay pure (no Drift/Firestore import); grep-guarded in Task 1 acceptance. |
| T-07-01-03 | Denial of service | repository load | accept | A malformed Firestore exercises payload falls through to the bundled seed (existing try/catch pattern); the practice path never blocks. Cold-install offline floor preserved (PLAT-01). |
| T-07-01-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (pure Dart models + JSON). Audio package + its legitimacy gate live in Plan 07-02. |
</threat_model>
