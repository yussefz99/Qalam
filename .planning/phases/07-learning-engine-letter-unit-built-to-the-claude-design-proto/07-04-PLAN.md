<objective>
Build the 5 reusable components — the whole engine UI — pixel-faithful to the Claude Design
prototype: `ExerciseScaffold` (the RTL landscape page shell), `PromptHeader` (the ordered PromptPart
renderer — the composition engine), `WriteSurface` (a thin config wrapper over the EXISTING
`StrokeCanvas`), `FeedbackPanel` (pass = one quiet star + praise; fix = coral + specific authored
line), and `ProgressRibbon` (R→L position dots). An `ExerciseController` drives the
idle→think→pass/fix state and the mascot pose from the `validateExercise` result. A `QalamTokens`
class maps the kit `:root` 1:1 (with the two name drifts reconciled: `--gold`→`--gold-ink`,
`--white`→`--surface-raised`).

`components.css`/`components.js` structure 1:1; the ink/trace primitive, mascot (`QalamMascot`, already
has all 5 poses), and tokens are REUSED. A new question = a new config, never new UI.

Purpose: every unit section (Plans 07-05, 07-06) renders by feeding these components a config — they
build no bespoke UI. Output: the 5 components + controller + tokens + ARB copy + widget tests proving
all 6 question types render config-driven.
