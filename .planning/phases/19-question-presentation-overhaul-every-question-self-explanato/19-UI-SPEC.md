# Phase 19 ŌĆö UI Design Contract

> Visual and interaction contract for the question-presentation overhaul. Every non-trace
> question must be self-explanatory from the screen alone ŌĆö a persistent instruction bar,
> a large stimulus zone, and a per-type "what to do" affordance, with the spoken line as
> reinforcement, never the only carrier.
>
> Generated by gsd-ui-researcher (AUTO MODE ŌĆö owner delegated visual treatment within the
> design kit per CONTEXT.md; owner signs the ~10 instruction strings later). Verified by
> gsd-ui-checker.

**Design system source of truth:** `docs/design/kit/project/colors_and_type.css` (`:root`
tokens), mirrored 1:1 in Dart at `lib/theme/qalam_tokens.dart` / `lib/theme/colors.dart`
(semantic layer: `QalamColors` / `QalamSpace` / `QalamRadii` / `QalamShadows` /
`QalamMotion` / `QalamTextStyles`). Brand rules: `docs/design/kit/project/SKILL.md`.
**No hex or font is hard-coded ŌĆö every value below cites an existing token.**

## Spacing Scale

Project scale is a **4px base** (`colors_and_type.css` `--space-*`), mirrored in `QalamSpace`.
All declared values are multiples of 4.

| Token | Value | Usage this phase |
|-------|-------|------------------|
| space-1 | 4px | icon inner gaps, hairline offsets |
| space-2 | 8px | compact chip padding |
| space-3 | 12px | iconŌåölabel gap in the instruction bar / audio card |
| space-4 | 16px | **minimum gap between touch targets** (kids-UX floor); default element spacing |
| space-5 | 20px | stimulus-word run spacing |
| space-6 | 24px | card padding (instruction bar, audio card ŌĆö new surfaces) |
| space-8 | 32px | stimulus-zone vertical rhythm |
| space-12 | 48px | major section breaks |
| space-16 | 64px | **minimum touch-target height** (kids-UX floor) |

**Touch-target floors (kids-UX, SKILL.md ŌĆö hard rule, not optional):**
- Minimum interactive target: **64px** (`--target-min`)
- Comfortable: **72px** (`--target-comfy`)
- Hero (audio card): **96px** (`--target-large`)
- Minimum gap between two targets: **16px** (`--space-4`)

**Exceptions (documented):**
- The `writeWord.copy` card and the slot-word card **extend the shipped `_TextPart`** and
  inherit its established `22px` horizontal padding for pixel continuity with the existing
  PromptHeader cards. This is a **legacy exception only** ŌĆö any NEW surface built on this
  pattern rounds to the grid (20px `space-5` or 24px `space-6`) going forward; 22px must not
  propagate beyond the two inherited cards.
- Control radii `14/20/28` are the named kit radii (`--radius-md/-lg/-xl`), not spacing.

## Typography

Phase-scoped roles (each maps to an existing `QalamTextStyles` role + kit `--fz-*` token).
Sizes used: **40 / 20 / 16 / 14**. Weights used: **exactly two** ŌĆö **500** (Fredoka, all
English UI roles incl. muted captions) and **600** (Nunito semibold body/labels + Arabic
emphasis). Muted helper captions stay at weight 500 and differentiate via `fg-muted` color,
never via a third weight.

| Role | Element | Size | Font / Weight | Line Height |
|------|---------|------|---------------|-------------|
| Display (Arabic stimulus) | The word-to-copy / slot-word / stimulus glyph | 40px (`--fz-ar-large`) | Noto Naskh Arabic, 600, deep-ink | 2.0 (`--lh-tashkeel`, all content is vocalized) |
| Heading (instruction) | Instruction-bar per-type text | 20px (`--fz-20`) | Fredoka, 500, fg | 1.3 (`--lh-snug`) |
| Label (affordance) | "I'm Ready" / "Peek" / "Hide" / audio-card "Listen" | 16ŌĆō20px (`--fz-16`/`--fz-20`) | Fredoka, 500 | 1.3 (`--lh-snug`) |
| Body (helper) | Muted caption (e.g. tap-to-hear semantics text if shown) | 14px (`--fz-14`) | Nunito, 500, fg-muted (color carries the muting, not weight) | 1.5 (`--lh-body`) |

**Arabic rules (SKILL.md, non-negotiable):** wrap in RTL (`Directionality.rtl` / `ArabicText`
island); 10ŌĆō25% larger than nearby English; line-height 1.7 plain, **2.0 with tashkeel**;
never bold, never italic (weight variation only ŌĆö 600 max); Western numerals inside Arabic
blocks LTR-isolated (`.q-num`).

**Case convention (deliberate, documented):** the instruction bar uses **sentence case**
("Trace the letter") ŌĆö it is a read-aloud imperative to a 5ŌĆō10-year-old, one idea per line,
not chrome. True buttons keep **Title Case** ("I'm Ready", "Peek", "Listen") per SKILL.md.

**Accent (`ink-teal`) reserved for ŌĆö explicit list, nothing else:**

**Destructive / semantic-negative:** none in this phase. The v6ŌåÆv7 keying migration is

## Component & Interaction Contract (phase-specific)

The single render path is `ExerciseScaffold` (all ~10 types) with `PromptHeader` composition.
Do **not** add a bespoke widget tree per type; extend the scaffold (RESEARCH anti-pattern).
Every new surface mounts through `presentGraphExercise(presentEpoch:)` (fresh-mount contract,
18-12 ŌĆö do not break).

### 1. Instruction bar (D-01/D-02/D-03) ŌĆö `ExerciseScaffold._mainColumn`

| Property | Contract |
|----------|----------|
| Placement | A fixed strip between the ribbon row and `PromptHeader` (`_mainColumn`). Same position on every graded type. Guarded by the existing `_hasInstruction` (excludes `teachCard`). |
| Composition | leading icon (24px, ink-teal) ┬Ę per-type text (20px Fredoka 500, fg) ┬Ę trailing speaker glyph (24px, ink-teal) = the replay affordance |
| Surface | `--teal-tint` fill, `--aqua-edge` 1.5px border, radius 16 (`--radius-md`ŌåÆ use 16), min-height **64px**, horizontal padding 24, iconŌåötext gap 12 |
| Content source | Per-type template map keyed on `exercise.type` (D-02) ŌĆö **NOT** a transcript of the `say` line (Pitfall 6: `say` stays filtered out of `PromptHeader`; the bar never renders `SayPart`) |
| Tap behaviour | Whole bar is one tap target ŌåÆ re-invokes the existing `_speakInstructionThenRelease()` verbatim (stops in-flight voice, arms the 8s-capped instruction-hold, no-ops on empty/teachCard). **Absorbs `_HearAgainCta`** ŌĆö the old pill (`exercise_scaffold.dart:730-739, 1089`) is deleted; ONE replay affordance, never two (Phase-07 double-Hear-button precedent, D-03) |
| Auto-speak | The `say` line still auto-speaks once on question start (18 instruction-hold unchanged: canvas dimmed to 0.45 + `IgnorePointer` while speaking, 8s cap). The **bar is never dimmed** ŌĆö only the canvas is held |
| States | **default** (teal-tint strip); **pressed** (InkWell press, no color change to gold); **speaking** (optional gentle pulse on the trailing speaker glyph, `--dur-base` 220ms `--ease-out-quart`, no gold); **hidden** (teachCard / empty say-line) |
| Semantics | `Semantics(button: true, label: "Hear it again")` on the bar; the per-type text is exposed as readable content |
| Anti-pattern | Do NOT place the persistent instruction into the mascot speech bubble ŌĆö the bubble is reserved for the coaching voice (D-01) |

### 2. Audio stimulus card (D-07) ŌĆö `PromptHeader._AudioPart`, hero variant

| Property | Contract |
|----------|----------|
| Role | The "sound to write" ŌĆö a large tappable card filling the stimulus zone (visually distinct from the bar's light replay strip) |
| Surface | `--ink-teal` fill, white speaker icon **40px** + "Listen" label (20px Fredoka 500, white), radius 28 (`--radius-xl`), min-height **96px** (`--target-large`), padding 24, sticker shadow `0 4px 0 --deep-ink` |
| Behaviour | Auto-plays the clip once on mount (mirrors the 18-12 auto-speak); tap replays any time |
| Placeholder state (D-07/D-11) | If the 18.1 partner clip asset is absent, the card **still renders**; tap falls back to the existing TTS say-line seam. **Silent-degrade** ŌĆö no error surface, no broken-audio icon (project pattern) |
| States | **idle/ready**; **playing** (gentle pulse on the speaker glyph, `--dur-base`, no gold); **pressed** |

### 3. Gap slot box (D-06) ŌĆö `PromptHeader._GapWord` / `_GapLetter`, enlarged

| Property | Contract |
|----------|----------|
| Rendering | The word renders at **full stimulus size (40px Arabic, RTL-correct)** on a `--surface-raised` card; the missing piece is a **big highlighted slot box** exactly where it belongs in reading order. The literal `__blank__` / `_letter_` marker never leaks to screen (Pitfall 6 + test: assert no `__blank__` text finder) |
| Slot visual | Rounded box, radius 14, **2px `--ink-teal` outline**, min-height ~64px (matches the 40px glyph line), min-width 72px (word slot) / 56px (letter slot). **Chosen affordance = gentle pulse** (D-06 offered dotted OR pulse): opacity/scale pulse `--dur-slow` 420ms `--ease-in-out`, so the eye lands on the gap. (Dotted border is the discretionary alternative; pulse is chosen to avoid a dashed-border painter and match the shipped teal-ring rendering.) |
| RTL | Word + slot laid out RTL (`Directionality.rtl`, existing `Wrap`); the slot sits at the correct position for the missing letter/word |
| States | **default** (teal outline + slow pulse); the box is a display target, not tappable |

### 4. Copy-word hide + peek (D-05) ŌĆö new `copy_stimulus.dart` (small StatefulWidget)

| Property | Contract |
|----------|----------|
| Revealed | The word shows **large (40px Arabic)** on a `--surface-raised` card + an **"I'm Ready"** button (Title Case, min-height 64px) beneath |
| Hide trigger | Child taps "I'm Ready" **or** starts the first stroke ŌåÆ the word hides (card blanks to a calm placeholder). Replaces today's static `opacity 0.18` dim (`prompt_header.dart:372`) |
| Peek | A **"Peek"** button (min-height 64px, Ōēź16px from siblings) re-reveals the word; while revealed a **"Hide"** toggle returns it. **Nothing vanishes on a timer** (D-05) ŌĆö every reveal/hide is child-controlled |
| Memory intent | Recall stays honest ŌĆö the peek is a deliberate, child-initiated assist, not an auto-fade |
| States | **revealed** ┬Ę **hidden** ┬Ę **peeking** (toggle back to hidden via "Hide") |

### 5. Recall write types (D-08) ŌĆö absence rule

| Property | Contract |
|----------|----------|
| Rule | `writeLetter` / `writeWord` **recall** questions show **no letter model** (no ghost, no faint glyph). The Phase-18 remediation arc IS the only hint path (same-criterion fail streak steps down to trace). One hint mechanism, not two |
| Enforcement | Data/widget test: assert no model/ghost part is authored on recall configs (QP-06) |

### 6. Micro-drill re-add (D-18) ŌĆö `SpotlightOverlay` (18-10), no new visuals

| Property | Contract |
|----------|----------|
| Chrome | Reuse the shipped `SpotlightOverlay` (dot/bowl/start zones, `IgnorePointer`-safe). No new overlay |
| Instruction bar | Shows "Practice this part" with optional per-criterion override: `dot`ŌåÆ"Practice the dot", `shape`ŌåÆ"Practice the curve", `strokeOrder`ŌåÆ"Practice the start" |

**Motion (all new surfaces):** gentle only ŌĆö `--dur-fast` 140ms / `--dur-base` 220ms /
`--dur-slow` 420ms with `--ease-out-quart` (or `--ease-in-out` for the slot pulse). No bouncy
