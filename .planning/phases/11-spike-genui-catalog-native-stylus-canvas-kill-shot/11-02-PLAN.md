<objective>
Build the throwaway A/B harness that IS the kill-shot: register the real native StrokeCanvas as
a custom GenUI CatalogItem and embed it — under a stable ValueKey — inside a model-generated tree
where GenUI also renders one coaching line above it (the genuinely mixed tree, D-03/D-04). Drive
it with a Gemini Flash model via Firebase AI Logic over GenUI's A2UI streaming transport, calling
ONE present_activity catalog item. Wire a toggle so the same baa trace can be felt on the
GenUI-embedded canvas [A] versus the identical standalone canvas [B] (D-05). All additive, in its
own main_spike_genui.dart target; production boot stays untouched.

Purpose: this is the seam Phase 14's GATE decision turns on. If the embedded canvas keeps its
stroke State across GenUI surface rebuilds (stable key) and the pen stays native + lag-free, the
GATE can be "keep". If GenUI's reactive rebuild tears the canvas State down or steals the stylus
drag, that IS the "drop GenUI" evidence (Pitfalls 1 & 2).
Output: a runnable spike app with both arms wired, ready for the on-device A/B in Plan 03.
</objective>
