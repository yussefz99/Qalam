<objective>
Close the BLOCKER stuck-state gap (UAT test 6) and the retry-no-op gap (UAT test 3) with ONE coordinated fix. Both share the exact same mechanism per both debug sessions: `exercise_presenter.dart` keys `ExerciseScaffold` by `ValueKey('graph:$exerciseId')` alone, so re-presenting the SAME graph-node id (a legitimate retry-in-place on a first fail, OR an active-arc re-present of the floor-trace across entry/stepDown/rebuild) makes Flutter reuse the existing Element instead of remounting it. `_ExerciseScaffoldState.initState()` — the ONLY place that resets the controller phase to idle, clears the canvas, and re-arms the instruction-hold/TTS — never re-runs, so the tap is a silent no-op. On the fail path the child still has a working "Clear"; on the active-arc PASS path the only CTA is a single "Next exercise" button, so every tap is a permanent no-op and only a force-quit recovers.

Purpose: A single wrong attempt or an arc re-present must NEVER strand the child. A fix that only covers the first-fail case (test 3) will not fix the more severe stuck-with-no-escape case (test 6) — both need the same underlying support for re-presenting the same node id.

Output: A presentation-epoch counter folded into the presenter key so every advance forces a fresh mount (resetting phase, canvas, and instruction-hold), plus an always-available control to replay the spoken instruction (the secondary ask in test 3's report).
</objective>
