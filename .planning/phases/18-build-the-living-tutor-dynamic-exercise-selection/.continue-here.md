## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Worktree cleanup-wave SUMMARY.md rescue collision | The SDK's `worktree cleanup-wave` helper unconditionally re-copies each worktree's `*-SUMMARY.md` into the main tree as an untracked file right before merging, even when the executor already committed it — git then refuses the merge ("untracked working tree files would be overwritten"). Retrying the same command just re-triggers it. | advisory | Delete the stray untracked `*-SUMMARY.md` from the main tree, then merge each worktree branch manually (`git merge <branch> --no-ff --no-edit -m "..."`) instead of relying on `worktree cleanup-wave` when executors have already committed their SUMMARY. |
| Auto-mode classifier hard-blocks `gcloud run deploy` | Even with explicit prior owner authorization ("fire whatever you need immediately"), the Claude Code auto-mode classifier refuses production deploy commands outright — this is not a permission prompt, it's a hard denial. | advisory | The owner must exit auto mode for that one action (or run the deploy command themselves); retry the identical command once auto mode is off — it then surfaces a normal approval prompt. |
| `flutter run` over USB can install-then-SIGKILL if the device is locked | First `flutter run -d <ipad>` attempt built fine, installed, then the process got SIGKILLed immediately after launch and hung forever on "Dart VM Service was not discovered." | advisory | Confirm the iPad is unlocked with the screen on before running; if it hangs post-install with no progress for a while, kill the task and retry — a clean retry with the device awake succeeded in under a minute. |
