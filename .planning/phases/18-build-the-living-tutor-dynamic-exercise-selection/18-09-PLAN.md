<objective>
Build the across-session half of the two-timescale model: a nightly Python job that aggregates each
child's per-letter×criterion evidence into a per-criterion EMA and writes a derived-only, non-PII
`child_models/{uid}` profile doc (D-15/Req 8). It runs as a Cloud Run Job in the existing server
image (reusing firebase_admin + ADC — zero new package). Letter-agnostic by construction: a newly
signed letter appears in evidence and compiles with no code change, proven by a second-letter fixture.

Purpose: the compiled `strengths[]`/`struggles[]` are what the next session reads (via the Drift
mirror, 18-06) so a returning child's first pick reflects the past (Req 2).
Output: compile_profiles.py entrypoint + tests (multi-letter, second-letter, PII guard), Job-runnable.
</objective>
