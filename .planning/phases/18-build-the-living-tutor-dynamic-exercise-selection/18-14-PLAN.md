<objective>
Close the STRUCTURAL half of the static-feedback gap (UAT test 5). The recurring line the owner sees ("your baa wants a deeper bowl…") is NOT the LLM echoing an exemplar — it is a client-side fallback that fires because `TutorInsight.rationale` is structurally null on almost every clean pass. The debug traced the root cause to a tool-schema gap: `COACH_NEXT_EXERCISE_ADDENDUM` already ASKS the model to attach `nextExerciseId` + a `rationale` argument, the coach node already rails those args, `main.py` already renames `next_exercise_id`→`nextExerciseId`, and `remote_agent_brain.dart` already parses them into `TutorPlan` — but the 4 ACTION tools in `server/app/tools.py` do NOT DECLARE `nextExerciseId`/`rationale` as parameters, so a real Gemini/Vertex function-calling model CANNOT attach undeclared keys. The whole per-attempt WHY chain is dead at its first link.

Purpose: Make the coach's real, per-attempt rationale reach the client on the common clean-pass path so the Teacher's Margin can show a generated WHY (the client-side presentation + fallback-variance is closed separately by 18-16, which reads this `rationale`).

Output: The ACTION tools declare `next_exercise_id` + `rationale` as optional parameters (the schema-backed fix), a server schema test proves it, and a wire test proves a coach decision carrying them serializes to `CoachOut.args` in the camelCase form the Dart parser reads. The Cloud Run re-deploy is a terminal human gate (each prod deploy needs fresh explicit owner wording, per project convention).
</objective>
