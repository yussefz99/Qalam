## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /coach | untrusted body; the uid keying evidence must come from the verified ID token, not the payload |
| /coach → Firestore (Admin SDK) | server-side write with elevated Admin privileges; must write only derived non-PII rows |
| client → Firestore (rules) | client may READ its own profile doc; may write NOTHING |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-05-01 | Spoofing | evidence uid | mitigate | uid taken from verify_caller (Firebase ID token + App Check), never from the request body |
| T-18-05-02 | Tampering | client forges child model | mitigate | Firestore client writes stay deny-all (D-13); the model/evidence written only by the Admin SDK, which bypasses rules |
| T-18-05-03 | Elevation of privilege | account A reads account B's model | mitigate | `match /child_models/{uid}` owner-only read (request.auth.uid == uid); evidence subcollection has no client match |
| T-18-05-04 | Information disclosure | new wire field smuggles PII/geometry | mitigate | ChildProfileIn/EvidenceDigestRowIn are extra=forbid, fixed-vocabulary; test_schema_forbid rejects a nested coordinate key; server ships FIRST (422 lockstep) |
| T-18-05-05 | Denial of service (UX) | evidence write stalls the child | mitigate | Admin-SDK append is a BackgroundTask off the CoachOut critical path; batched writes; try/except-log so /coach never breaks |
| T-18-05-SC | Tampering | package installs | accept | No new package — firebase_admin.firestore already installed + ADC-initialized in auth.py (RESEARCH §Package Legitimacy Audit) |
</threat_model>
