# Phase 6 — UI Design Contract

> Visual and interaction contract for Lesson Progression & Home (S1-01, S1-09).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Source of truth for visuals: `docs/design/kit/` (Qalam Design System) + the in-code
> theme tokens (`lib/theme/`). This phase makes existing surfaces LIVE — Home today-card,
> Journey map, Mastery celebration, Practice feedback zone — it introduces **no new
> screens** and **no new tokens**.

---

## Scope Note (read first)

This phase is **Flutter / Android / RTL** — NOT a React/Next/Vite web app. The shadcn
initialization gate does not apply (`Tool: none`). The design system is already fully
codified in `lib/theme/` (one-way translation from `colors_and_type.css`):
`QalamColors`, `QalamSpace`, `QalamTargets`, `QalamRadii`, `QalamShadows`,
`QalamMotion`, `QalamInk`, `QalamFonts`, `QalamFontSizes`, `QalamTextStyles`.
**Widgets read these tokens — never raw hex, never magic numbers, never inline fonts.**

### Surfaces touched (existing → live)

| Surface | File | What changes visually |
|---------|------|-----------------------|
| Home today-card | `lib/screens/home_screen.dart` `_TodaysLessonCard` | Live letter glyph + title (D-08), ink-fill progress (D-09), prepared-desk entrance (D-13), all-mastered end state (D-11) |
| Journey map | `lib/features/journey/journey_screen.dart` | Live mastery data; skipped-but-unlocked letters reuse the "future" visual but become tappable (D-07); highlight arrival from celebration (D-15) |
| Mastery celebration | `lib/features/practice/widgets/mastery_celebration.dart` | Primary "Next Lesson" CTA (D-14), last-lesson variant (D-16), "show someone at home" tutor line (D-17) |
| Practice feedback zone | `docs/design/practice-redesign/` ShowFix zone | Slow-motion ghost comparison "Watch the difference" (D-21) |

The scaffolding fade (D-18/D-19/D-20) is **invisible to the child** — no "loose/strict"
labels, no difficulty chrome anywhere. It is a scoring contract, not a UI contract.

### Mandatory omissions (anti-gamification — PLAT-03, binding)

The demo ARB keys (`demoHomeStarCount`, `demoHomeThisWeek`, `demoCelebrationStarsDelta`,
etc.) and the owner's original mockups predate the anti-gamification decision. They are
demo-only. On the REAL surfaces this phase touches:

| Forbidden | Rule |
|-----------|------|
| Running star counter / total anywhere | Stars are per-letter mastery markers only |
| "THIS WEEK" tally, weekly stats, streaks, badges | OMIT |
| "+N today" / "keep going" hype copy | OMIT |
| "N of 28 mastered" as a SCORE treatment | Allowed only as quiet information (existing journey header), never gold, never celebrated |
| Draft-letter markers visible to the child | D-04 — sign-off state is the adults' `/dev` view only |
| Level-2 teasing on the all-mastered card | D-11 — factual, calm, no hype |

## Spacing Scale

Declared values (4px base — `QalamSpace`; all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| space1 | 4px | Hairline gaps, inline padding |
| space2 | 8px | Compact element spacing (card title ↔ subtitle) |
| space3 | 12px | Tight stacks |
| space4 | 16px | Default gap; **minimum gap between touch targets** (kids-UX floor) |
| space6 | 24px | Card inner gaps / section padding |
| space8 | 32px | Card padding (`_TodaysLessonCard` keeps `space8`), group gaps |
| space12 | 48px | Major section breaks (celebration CTA block) |
| space16 | 64px | Glyph container on the today-card (existing) |

**Touch-target floors (`QalamTargets` — binding):** interactive elements ≥ **64px**
(`targetMin`); the today-card chevron pill and all celebration buttons ≥ 72px
(`targetComfy`). Journey nodes keep their existing sizes (Phase 03.1). Minimum 16px
between targets.

Exceptions: the ink-fill glyph and journey star badges are non-interactive visual
indicators, sized for legibility not for the 64px floor. One task per screen
(kids-UX): Home keeps exactly ONE primary action — the today-card (D-12).

## Typography

Two **script-isolated** scales, as established in 03-UI-SPEC: an English scale (LTR
chrome) and an Arabic scale (RTL content islands). This phase introduces **no new type
roles** — every new element maps to an existing `QalamTextStyles` role.

### English type scale — exactly 4 roles for this phase's surfaces

| Role | Size | Weight | Line Height | Font | Used for (this phase) |
|------|------|--------|-------------|------|----------------------|
| Heading | 28px | 500 | 1.3 | Fredoka | Today-card title ("The Letter Baa"), all-mastered card title, Home greeting (existing) |
| Title / Button | 24px | 500 | 1.0 (button) / 1.3 (panel title) | Fredoka | "Next Lesson" / "Back Home" / "See Journey" button labels; "Watch the difference." ghost-comparison panel title |
| Body | 18px | 400 | 1.5 | Nunito | Card subtitle, all-mastered body, the D-17 tutor line |
| Label | 16px | 600 | 1.3 | Nunito | "TODAY'S LESSON" eyebrow, "Yours" / "Qalam's" comparison labels, journey pills |

> **4-size scale for this phase: 28 → 24 → 18 → 16.** Two weights per family:
> Fredoka 500 (+600 only on the pre-existing 42px Display), Nunito 400/600.
> The 42px Display role exists on the celebration heading ("You learned alif.") —
> that element is **unchanged by this phase** and stays at its Phase-3 contract;
> Phase 6 adds only buttons (24px) and a body line (18px) to that screen.

### Arabic type scale — RTL islands only (distinct from the English cap)

Never bold, never italic, never letterSpacing (`letterSpacing: 0` explicit — Pitfall 2).

| Role | Size | Weight | Line Height | Font | Used for (this phase) |
|------|------|--------|-------------|------|----------------------|
| Arabic display | 96px | 500 | 1.1 | Cairo | The today-card letter glyph (live, via `ArabicText(display: true)`) — now also the **ink-fill progress surface** (D-09) |
| Arabic body | 26px | 400 | 1.7 | Noto Naskh | Arabic praise islands (existing, unchanged) |

Western numerals everywhere, LTR-isolated inside Arabic blocks (Phase-1 decision).

## Color

60 / 30 / 10 split using `QalamColors`:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAF6EE` `QalamColors.bg` (parchment) | Home background, journey background, celebration background — never stark white |
| Secondary (30%) | `#EAF4F4` `QalamColors.surface` (soft aqua) + `#DCEEEF` `primaryTint` | Today-card surface, nav rail, glyph container tint, journey node fills |
| Accent (10%) | `#168A8F` `QalamColors.primary` (ink-teal) | See reserved-for list below |
| Reward | `#F2A60C` `QalamColors.reward` (gold) | **REWARDS ONLY** — see exhaustive list below |
| Success | `#3FB984` `QalamColors.success` (leaf) | Journey completed-path overlay + complete-node treatment (existing) |
| Soft-warn | `#FF8A6B` `QalamColors.warnSoft` (coral) | The child's wobbly stroke in the ghost comparison (D-21) — the only "error" color; never red |
| Ink | `#0E5B5F` `QalamColors.inkStroke` (deep-ink) | The ink-fill ramp target (D-09), reference stroke in the ghost comparison |

**Accent (teal) reserved for** (exhaustive): the today-card chevron pill, the
"Next Lesson" primary CTA, the active nav-rail item, the journey current-node ring +
TODAY pill, the "MASTERED" badge. NOT for secondary actions — "Back Home" (demoted)
and "See Journey" use ghost/outlined treatment on parchment.

**Gold reserved for** (rewards only, exhaustive for this phase): journey star badges
on complete nodes (existing), the single celebration star (existing), the D-15
highlight-arrival star settling onto the just-mastered node, the gold start-dot in
practice (existing). **The ink-fill is NOT gold** — it is deep-ink on the parchment
palette (D-09: "pure design-system rendering, no gamification"). If gold appears,
a mastery just happened.

**Ink-fill ramp (D-09, prescriptive):** the today-card glyph renders in deep-ink
(`QalamColors.inkStroke`) at opacity `0.25 + 0.75 × (persistedCleanReps / cleanRepsToAdvance)`
— 0 reps = a faint 25% ink wash, each clean rep deepens the shade equally, mastered =
solid deep-ink. Carry a `Semantics` label "{n} of {N} clean reps"; **no visible numeric
rep text on the card** (the ink IS the progress — it replaces rep-dots, D-09).

**Destructive:** none — this phase has no destructive actions.

## Copywriting Contract

All user-facing copy lives in `AppLocalizations` (gen-l10n) — never hardcoded. Voice:
warm, calm, specific; second person to the child; Title Case on buttons; the tutor
speaks plainly, never chatbot-cheerful. A little Arabic is welcome; guidance stays in
English. New ARB keys below; existing keys (`homeGreeting`, `homeLessonEyebrow`,
`practiceCelebration*`) are reused unchanged.

### Home today-card (live)

| Element | Copy | Notes |
|---------|------|-------|
| Eyebrow | `TODAY'S LESSON` | existing key `homeLessonEyebrow`, unchanged |
| Title | `The Letter {letterName}` | NEW key `homeLessonTitleFor` with `{letterName}` placeholder (romanized, e.g. "Baa"); replaces static `homeLessonTitle`. Lesson-title strings inside `lessons.json` may use the "Lesson N — {Letter}" pattern (content, not UI copy) |
| Subtitle | `Stroke order and tracing` | existing key `homeLessonSubtitle`, unchanged |
| Primary CTA | The whole card is the Start (tap → `/practice?lesson={id}`); the teal chevron pill is the affordance. **Keep the existing layout** (D-08) — no extra "Start" button | S1-01: one clear Start, no navigation required |

### Home all-mastered end state (D-11)

| Element | Copy |
|---------|------|
| Eyebrow | `YOUR LETTERS` |
| Title | `You've mastered all your letters.` |
| Body + next step | `Visit your journey to practice any letter again.` |
| CTA | Card tap → `/journey` (chevron pill kept; review practice happens from journey green nodes, D-12) |

Factual and calm — no "amazing!!", no Level-2 teaser, no totals.

### Loading / error (today-card provider states)

| State | Treatment |
|-------|-----------|
| Loading | Card renders with the glyph container empty and title area blank — no spinner chrome, no skeleton shimmer (mirrors the `_GreetingHeader` degradation pattern) |
| Error | Degrade to the profile's `startingLessonId` lesson (default alif) so the child always has a Start; shell-level failures reuse existing `databaseError` copy. Never show a raw error to the child |

### Mastery celebration additions (D-14 / D-16 / D-17)

| Element | Copy | Notes |
|---------|------|-------|

| Secondary | `Back Home` | existing key, **demoted to ghost/outlined** treatment (one primary per screen) |
| Tertiary ghost link | `See journey` | existing key `journeySeeJourney`, unchanged; navigates `/journey?highlight={masteredId}` (D-15) |

### Ghost comparison (D-21)

| Element | Copy | Notes |
|---------|------|-------|
| Trigger (ghost button in ShowFix zone) | `Watch the Difference` | NEW key `ghostCompareButton`. Title Case, ghost treatment beside the existing `Show Me Again` |

## Interaction & Motion Contract

| Moment | Behavior | Tokens / Source |
|--------|----------|-----------------|
| Home opens — prepared desk (D-13) | The today-card settles in like a teacher laying out a worksheet: card slides up ~24px + fades in with `easeOutQuart` over `durSlow` (420ms); the letter glyph then fades up over `durBase` (220ms). Plays **once per arrival** at Home (not on rebuilds). If `MediaQuery.disableAnimations` is true, render settled immediately | `QalamMotion.easeOutQuart`, `durSlow`, `durBase` |
| Ink-fill (D-09) | Static per render — the glyph shows the persisted rep-depth (no animation needed when reps were earned in a previous session). When Home is re-entered after a rep was just earned, the deeper shade simply renders; no celebratory transition (the celebration screen owns that moment) | deep-ink opacity ramp (Color contract above) |
| Today-card tap | Whole card → `/practice?lesson={todayLessonId}` (route param mechanics are executor discretion per CONTEXT) | existing `todaysLessonCard` key |
| Pass → unlock (D-14) | "Next Lesson" goes directly into the newly unlocked letter's practice flow; returning Home shows the new lesson as today's | S1-09: unlock is immediate on pass |
| Celebration → journey highlight (D-15) | Arriving via `/journey?highlight={id}`: the just-mastered node's gold star badge **settles in** (scale 0→1, `easeSoftBack`, `durCheer` 700ms — same recipe as the celebration star), then the pulse glow moves to the new current node. One-shot; no sound | `QalamMotion.easeSoftBack`, `durCheer`; gold = reward only |
| Journey nodes (D-07) | Complete + current + **skipped-but-unlocked** nodes are tappable → `/practice?lesson={id}`; locked/future-locked nodes stay inert. Skipped nodes keep the existing future visual (white, dashed border) — no new visual state | existing Phase-03.1 node states |
| Ghost comparison (D-21) | Tapping "Watch the Difference" plays the child's wobbly stroke (coral) beside the reference stroke (deep-ink) at **half speed** — `durWrite × 2` = 2800ms, linear curve (even pen speed). Replayable. **In-memory strokes only — stroke points are NEVER persisted** (T-03-01) | `QalamMotion.durWrite`; `warnSoft` vs `inkStroke`; reuse `StrokeOrderAnimation` path machinery |
| Locked lessons (S1-09) | Visibly unavailable only on the Journey (existing locked/future visuals). Home never lists locked lessons — it shows exactly one prepared lesson | existing journey states |

RTL note: app chrome stays **LTR** (English). Arabic appears only as RTL content
islands via `ArabicText` (the card glyph, praise lines). Mirror only Arabic content
blocks, never the whole app.
