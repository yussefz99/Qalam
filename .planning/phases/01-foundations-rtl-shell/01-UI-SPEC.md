# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundations & RTL Shell phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Stack note:** This is a **Flutter (Dart) Android-tablet** project, not React/Next/Vite. The shadcn gate is **not applicable** — the design system is the bundled **Qalam Design System** (`docs/design/kit/`), the canonical source of truth. Every value below is **lifted from `docs/design/kit/project/colors_and_type.css`** (tokens) and `project/SKILL.md` + `project/assets/ICONOGRAPHY.md` (brand hard-rules), then mapped to a Flutter `ThemeData` + `ThemeExtension` layer (`lib/theme/`) per CONTEXT.md D-01.
>
> **Phase scope reminder:** Foundation + stylus de-risking spike. The surfaces to lock are the **theme/token layer**, **Arabic-capable typography**, **RTL conventions**, **three placeholder screens at shell/scaffold level only** (Home, Practice, Settings), and the **ink-stroke rendering treatment** for the Practice spike. The real tracing UI, dotted letters, tutor feedback, curriculum, and journey-map content are **out of scope** (P2–P9).

## Spacing Scale

Lifted from the kit's 4px base scale (`--space-*`). All values are multiples of 4. Map each to a Dart const in `dimens.dart`.

| Token | Value | Kit var | Usage |
|-------|-------|---------|-------|
| space1 | 4px | `--space-1` | Hairline gaps, inline padding |
| space2 | 8px | `--space-2` | Compact element spacing |
| space3 | 12px | `--space-3` | Tight stacks |
| space4 | 16px | `--space-4` | **Default gap; minimum gap between touch targets** |
| space5 | 20px | `--space-5` | Comfortable element spacing |
| space6 | 24px | `--space-6` | Card / section padding |
| space8 | 32px | `--space-8` | Layout gaps |
| space10 | 40px | `--space-10` | Large layout gaps |
| space12 | 48px | `--space-12` | Major section breaks |
| space16 | 64px | `--space-16` | Page-level spacing |
| space20 | 80px | `--space-20` | Hero / screen margins |
| space24 | 96px | `--space-24` | Hero / celebration spacing |

**Touch targets (kids-UX floor — SKILL.md hard rule, carry into the theme now):**

| Token | Value | Kit var | Usage |
|-------|-------|---------|-------|
| targetMin | 64px | `--target-min` | Minimum interactive target (children, stylus + finger) |
| targetComfy | 72px | `--target-comfy` | Comfortable target (nav, primary actions) |
| targetLarge | 96px | `--target-large` | Hero target (main Start-style actions, later phases) |

**Radii (`--radius-*`) — tactile, rounded, nothing sharp:** sm 8, md 14, lg 20, xl 28, 2xl 36, pill 999. Map all to `dimens.dart`.

Exceptions: Touch-target minimums (64/72/96px) are sizing floors, not spacing-scale values — they are intentionally large per the kids-UX rule and are an allowed exception to "spacing is for gaps." Interactive elements must be **≥64px and ≥16px apart**.

## Typography

Lifted from the kit type scale. **Chrome is English/LTR; Arabic is RTL content islands only** (CONTEXT D-05). Kids-UX floor: **18px base body**. Arabic sits 10–25% larger than nearby English. Phase 1 wires the full scale into `ThemeData.textTheme` (D-04), but the placeholder screens only exercise the roles below.

**English UI (LTR) — required roles for Phase 1 shell:**

| Role | Size | Family / Weight | Line Height | Kit source |
|------|------|-----------------|-------------|------------|
| Body | 18px | Nunito / 400 regular | 1.5 | `.q-body` / `--fz-18` `--lh-body` |
| Label | 16px | Nunito / 600 semibold | 1.3 | `.q-label` / `--fz-16` `--lh-snug` |
| Button | 24px | Fredoka / 500 medium | 1.0 | `.q-button` / `--fz-24` |
| Heading (H3) | 28px | Fredoka / 500 medium | 1.3 | `.q-h3` / `--fz-28` `--lh-snug` |
| Display (H1) | 42px | Fredoka / 600 semibold | 1.15 | `.q-h1` / `--fz-42` `--lh-tight` |

> Full scale to seed into the theme (used by later phases): `--fz-12/14/16/18/20/24/28/34/42/56`. Phase 1 declares them all in `text_styles.dart`; the shell renders only the roles above. Two English weights are in active use this phase — **400 regular** (Nunito body) and **600 semibold** (Nunito label, Fredoka display); Fredoka **500 medium** is the heading/button weight. Keep weight usage disciplined: do not introduce extra weights.

**Arabic content islands (RTL) — wired now, exercised in the glyph audit (D-12):**

| Role | Size | Family / Weight | Line Height | Kit source |
|------|------|-----------------|-------------|------------|
| Arabic body | 26px | Noto Naskh Arabic / 400 | 1.7 (plain) / **2.0 (with tashkeel)** | `.q-ar` / `--fz-ar-body` |
| Arabic large | 40px | Noto Naskh Arabic / 400 | 2.0 | `.q-ar-large` / `--fz-ar-large` |
| Arabic display | 96px | Cairo / 500 | 1.1 | `.q-ar-display` / `--fz-ar-display` (tracing-canvas glyph; spike reference) |

**Arabic hard rules (SKILL.md — non-negotiable, enforce in the theme/widgets):**
- Wrap every Arabic block in `Directionality(textDirection: TextDirection.rtl)` (the Flutter equivalent of `dir="rtl"`).
- Set Arabic 10–25% larger than adjacent English.
- Line-height **1.7 plain / 2.0 with tashkeel**.
- **Never bold, never italic** — weight variation only.
- All learning content is fully vocalized (tashkeel) — relevant when the glyph audit (D-12) uses real curriculum samples.
- **No Inter/Roboto** anywhere.

**Numerals (CONTEXT D-06):** **Western numerals (0–9) everywhere, including inside Arabic blocks.** Provide a `q-num`-equivalent: digits rendered in Nunito, tabular figures (`fontFeatures: [FontFeature.tabularFigures()]`), LTR-isolated inside RTL content. Decided explicitly — do not leave to locale defaults.

## Color

60/30/10 mapped from the kit's ink-and-parchment palette. **Background is warm parchment, never stark white** (CONTEXT D-02). All values map to semantic tokens in `colors.dart`.

| Role | Value | Kit var | Usage |
|------|-------|---------|-------|
| Dominant (60%) | `#FAF6EE` parchment | `--bg` / `--parchment` | App background on every screen; warm, never `#FFFFFF` |
| Secondary (30%) | `#EAF4F4` soft-aqua | `--surface` / `--soft-aqua` | Cards, the Practice spike canvas frame, nav rail, settings rows. Raised surfaces may use `#FFFFFF` (`--surface-raised`) sparingly. |
| Accent (10%) | `#168A8F` ink-teal | `--primary` / `--ink-teal` | See reserved-for list below |
| Reward (special, ≤rare) | `#F2A60C` gold-ink | `--reward` / `--gold-ink` | **REWARDS ONLY** — mastery stars / celebration. **NOT present in Phase 1** (no stars earned yet). Wire the token; never render it as a button or heading. |
| Success | `#3FB984` leaf | `--success` / `--leaf` | Correct / success affirmation (later phases) |
| Soft-warn (the only "error" color) | `#FF8A6B` coral | `--warn-soft` / `--coral` | "Let's try that again" — **never a harsh red, never a red X** |
| Text | `#222A2E` ink-charcoal | `--fg` | Body text — not pure black |
| Muted text | `#5C6B70` slate | `--fg-muted` | Secondary/caption text |
| On-primary text | `#FFFFFF` | `--fg-on-primary` | Text on ink-teal surfaces |

**Accent (ink-teal `#168A8F`) reserved for — explicit list (Phase 1 shell):**
- Primary action buttons / primary CTA (the placeholder "Open Practice" style action) and their pressed state (`--primary-pressed` deep-ink `#0E5B5F`).
- The active item in the placeholder nav (Home / Practice / Settings indicator).
- Screen-header / app-bar title accent.
- Focus / selected affordances on interactive elements.
- The rendered **ink stroke** on the Practice spike canvas (ink shade — see Ink Rendering below).

Accent is **not** used for: backgrounds, body text, card fills, decorative flourishes, or "all interactive elements" indiscriminately.

**Color hard rules (SKILL.md):**
- **Gold `#F2A60C` is rewards only.** If gold appears, something good just happened. Never a button, never a heading. (Absent in Phase 1.)
- **No red.** Errors are coral, framed warmly, with a soft wiggle — never a red X.
- **No purple-gradient AI slop.**

## Ink-Stroke Rendering (Practice spike surface)

The Practice screen is the stylus-spike surface. Phase 1 renders **live freehand ink only** (no dotted letter, no scoring — those are P3). The goal: rendered ink should read as **real ink on parchment**, consistent with "modern manuscript / playful calligraphy studio." Implement on a `CustomPainter` over a parchment canvas.

> **Kit precedent (`ui_kits/qalam_app/components.jsx` → `TracingCanvas`):** the inked stroke is drawn in **deep-ink `#0E5B5F`** with **`strokeLinecap="round"`**, no fill, on a `600`-unit viewBox at width `22` (≈18px at a 500px canvas). The dotted guide (P3, out of scope here) uses teal-wash `#B7DEDF`. Phase 1's live ink follows this precedent.

| Property | Value | Rationale |
|----------|-------|-----------|
| Canvas surface | parchment `#FAF6EE` (`--bg`), inset on a soft-aqua `#EAF4F4` framed card with `--radius-xl` (28px) and `--shadow-md` | Warm writing ground; matches the kit's framed tracing card |
| Stroke color | **deep-ink `#0E5B5F`** (`--primary-pressed`) at full opacity — matches the kit's inked stroke. (Ink-teal `#168A8F` `--primary` is the acceptable lighter alternative if calibration prefers it.) | The product's "ink"; reads as deliberate writing, on-brand |
| Stroke width | 6px baseline (tablet logical px), within a 4–8px range | Pen-like weight at tablet scale; thin enough for letter shapes, thick enough to feel like ink |
| Line cap | round | Soft, calligraphic ends — never hard squared ends (matches kit) |
| Line join | round | Smooth corners where stroke segments meet |
| Smoothing | quadratic/Catmull-Rom path smoothing between sampled points (not raw polylines) | Removes jitter; gives a flowing ink line. Keep latency low (spike must feel immediate). |
| Anti-aliasing | on (`isAntiAlias: true`) | No jagged edges |
| Pressure (if device reports it) | optional subtle width modulation within the 4–8px range | Nice-to-have; do not block the spike on pressure data |
| Palm/finger rejection | spike may capture all pointers, but **note** that P3 will filter to stylus-only via low-level pointer events | De-risk only; full rejection is P3 scope |

**Out of scope for the ink treatment in P1:** dotted guide letter, stroke-order animation, per-stroke scoring/feedback color, failing-stroke highlight, the quiet mastery star, confetti. Render the raw ink and a Clear action; nothing more.

## Screen Shells (scaffold level only)

Three placeholder routes via `go_router` (CONTEXT D-08). **Shell/scaffold only** — no real content. Chrome is **English/LTR**; landscape-locked (D-10).

| Route | Screen | Phase-1 content (placeholder) |
|-------|--------|-------------------------------|
| `/` | **Home** (journey-map placeholder) | App bar with قلم logo (Cairo wordmark via `flutter_svg`), parchment background, a centered placeholder card on soft-aqua stating the journey map arrives later, and a single primary action to open Practice. No stars, no totals, no streaks. |
| `/practice` | **Practice** (the stylus spike) | Parchment canvas (framed soft-aqua card), the live ink-rendering surface above, and a single **Clear** secondary action. This is the only screen with real interactive behavior in P1. |
| `/settings` | **Settings** (parent-area placeholder) | Parchment background, a list of placeholder rows; note a **routing seam** for a future PIN-gated `/parent/*` branch (built in P9) — do **not** build the PIN gate now. |

A minimal nav affordance (rail or app-bar tabs) switches the three routes; the active item uses the ink-teal accent. Keep **one primary task per screen** (SKILL.md kids-UX rule).

## Copywriting Contract

Voice (SKILL.md): speak **to the child** in second person, **Title Case** on buttons and headings, encouraging, short, one idea per line. Warm teacher, not a chatbot. **No emoji.** v1 ships **English strings only** (D-07) but routed through the `gen-l10n` seam — no hardcoded strings.

| Element | Copy |
|---------|------|
| Primary CTA (Home → Practice) | **Open Practice** |
| Practice action (clear the canvas) | **Clear** |
| Home placeholder heading | **Your Journey Starts Soon** |
| Home placeholder body | Your letters and lessons will live here. For now, head to Practice and write. |
| Practice empty state heading | **Write Here** |
| Practice empty state body | Pick up your pen and draw on the page. Tap Clear to start over. |
| Settings placeholder heading | **Settings** |
| Settings placeholder body | Grown-up settings will live here soon. |
| Error state (generic shell-level, e.g. database read failed) | **Something Didn't Save.** Close the app and open it again — your work is kept on this tablet. |
| Destructive confirmation — Clear canvas | **Clear your writing?** This erases what's on the page. → Buttons: **Clear** (coral-tinted, soft) / **Keep Writing** (default). No red, no red X. |

**Anti-gamification copy guardrails (CONTEXT D-13 / REQUIREMENTS PLAT-03):** no "+N", no "keep going!" hype, no streak/tally/badge language, no "Achievement unlocked." Stars (when they exist, later phases) mean **real mastery**, never a running score. Feedback, when it lands (P3), is specific — never "Oops, try again!"
